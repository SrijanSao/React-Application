Configuring Enterprise Manager User Accounts

Best practice: It is a best practice to create at least one additional Enterprise Manager user account that is a Local User, and to assign it to the Portal Administrator role. You can use this account to complete the remaining configuration steps if you choose. At this time, you can also create as many additional accounts as are needed for Enterprise Manager users in your environment and assign them to the appropriate Enterprise Manager roles.

For example, you can assign one or more Enterprise Manager users to the Portal Administrator role to grant administrator privileges to those users and enable them to see all Enterprise Manager pages. In contrast, assigning users to the Audit Administrator role grants access only to the Monitor and Logs pages. You can add as many Enterprise Manager users as you want to your environment, assigned to whichever roles are appropriate for each user.

Enterprise Manager users (who are responsible for administering and operating the Stealth environment) and Stealth endpoint users (who are usually authenticated using Active Directory/LDAP and are assigned to roles) exist in the same Stealth database, and so each user name must be unique. If a person in your environment acts as both an Enterprise Manager user and a Stealth endpoint user, he or she requires two distinct user accounts: an Enterprise Manager user account and an endpoint user account.

Adding Enterprise Manager Users

To add a new Enterprise Manager user, do the following:

  1. Select Administration in the menu bar.

  2. Select the User tab.

  3. Click Add at the bottom of the page.

  4. In the Add User dialog box, enter a unique display name for the new user in the Name box.

  5. Enter a unique user ID in the User ID box.

    Notes:

    • For a list of supported characters in user IDs, see Supported Characters and Length Restrictions in Enterprise Manager and Stealth(aware).

    • It is recommended that you add a prefix to each Enterprise Manager user name (in the User ID box) to ensure that there are no conflicts between Enterprise Manager users and endpoint users. For example, you could preface each Enterprise Manager user name with EM, such as EMOperator, EMAuditor, EMCrypto, and so on.

  6. Optionally, enter a description for the user in the Description box.

  7. Enter a password for the user in the Default Password box, and then reenter the password in the Confirm Password box.

    The password must be between six and 50 characters, and it must include all of the following.

    • At least one uppercase letter

    • At least one lowercase letter

    • At least one number

    • At least one of the following special characters:

      ! @ # $ % ^ & * ( ) _ + =

  8. Select the Local User check box.

  9. Click Save.

  10. When you are finished adding users, click Close.

Assigning Enterprise Manager Users to Roles

The Administration page, Role tab includes a list of roles to which you can assign to Enterprise Manager users. Assigning an Enterprise Manager user to a role determines which Enterprise Manager pages the user can access and tasks the user can perform. Available roles include the following:

  • Audit Administrator

    • Purpose – Manages logs and monitors the systems added to Enterprise Manager

    • Access – Monitoring page and Logs page only

  • Cryptographic Administrator

    • Purpose – Manages Stealth roles, COIs, and filters

    • Access – All pages, except the Administration page; however, users in this role are restricted from the following:

      • Users have read-only access to the Monitoring page and all subtabs

      • Users cannot view the License subtab on the Manage page, and cannot install or delete license files

  • Portal Administrator

    • Purpose – Performs all administration functions in Enterprise Manager

    • Access – All pages and functions

  • Portal Operator

    • Purpose – Monitors systems

    • Access – Monitoring page only

  • Service Administration

    • Purpose – Provides for automatic updates performed by services in the environment

    • Access – All pages and functions

  • Stealth Administrator

    • Purpose – Manages the settings for systems in the Stealth network

    • Access – Full access to the Monitoring and Logs pages and the System subtab on the Configure page

      Read-only access to the Provision page and all other subtabs (except System) on the Configure page

Click the plus sign (+) to the left of a role to display the users that have been assigned to the role.

To assign one or more Enterprise Manager users to a role, do the following.

Notes:

  • You cannot modify an Enterprise Manager user’s roles or delete the user while you are logged on as that user. You must sign in with another Enterprise Manager user account with the necessary privileges (for example, portaladmin).

  • You cannot modify the portaladmin user’s roles or delete the user.

  1. If you have not already done so, on the Administration page, select the Role tab.

  2. Select the check box next to the role or roles that you want to assign to a user.

  3. Click Show Users at the bottom of the page.

  4. In the Add/Remove User dialog box, select the check box next to each user that you want to assign to the selected roles.

  5. Click Save.

Importing Traffic Data (Import Traffic Data Tool)

Using the Import Traffic Data tool, you can import network traffic data from standard network analyzer tools (such as Wireshark and tcpdump) or from the Stealth Discovery Agent running on Nodes in the network.

Best Practice: It can take some time to complete the process of importing network traffic data. Understanding Time to Import Network Traffic Data to Stealth(aware) provides guidelines on network traffic data file processing time. To start working with the traffic data in Stealth(aware) more quickly, it is a best practice to copy a smaller amount of traffic data or a limited number of traffic data files to import at one time. It is recommended that you limit each pcap file you import to less than 30 MB, either by limiting the time you spend capturing data, or dividing one large file into multiple smaller files using a program like Wireshark editcap. It is also recommended that you incrementally import pcap files (one at a time) and that you complete the pcap file import for each capture point before advancing to another capture point.

To use the Import Traffic Data tool, do the following in Stealth(aware):

  1. Click the Import Traffic Data tool.

             

    The Import Traffic Data panel appears.

  2. From the Import Traffic Data panel, do one of the following to import new pcap or cap files or select a folder that includes pcap or cap files.

    Notes:

    • Only pcap or cap files can be imported (other file types, such as pcapng and NetFlow files in NFC format are not supported).

    • Files are not removed from the share when they are imported into Stealth(aware). Therefore, if you create a new project and browse to the same directory, all files that were previously imported into the old project can be imported into the new project.

    • When you import a new file, any empty Solutions in the model are deleted. To avoid this, add at least one Profile to the empty Solution.

    Click one of the following:

    • Click Browse to File, select a file to import, and then click Open.

    • Click Live Feed, browse to the file share you created in Configuring a Windows File Share for the Discovery Agent Network Traffic Data Files, and then click Select Folder.

      If you want to continuously receive updated data from one or more Discovery Agents you installed on Nodes, ensure that the Live Feed switch is On. The Live Feed automatically imports new pcap or cap files when they appear in the directory you have selected.

The Import Traffic Data panel closes, and a Processing message appears. The file or files are imported.

The Nodes are automatically classified and affinitized based on the affinity level set for the open project, as described in Setting the Affinity Level (Set Affinity Level Tool). The Processing message disappears when the data import process has been successfully completed.

You can import additional pcap or cap files to improve the information in the Stealth(aware) Model, and there is no limit to the number of files you can import. In addition, you can continuously update the model with new information from the Live Feed.

 Caution

Before updating the Stealth(aware) Model, ensure Live Feed is Off.

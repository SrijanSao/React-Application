Copying and Deploying the Project File to Enterprise Manager

Note: If Stealth(aware) is running on the Management Server, skip this topic.

If Stealth(aware) is running on a system other than the Management Server, you must transfer the project file you created to the Management Server. You must then run a PowerShell script to populate Enterprise Manager, and then—depending on whether you selected the Create Objects Only or the Create and Deploy option from the Deploy tool—either automatically or manually deploy the model onto your network. Do the following:

  1. If you have not already done so, copy the project file that you deployed from Stealth(aware) to any location on the Management Server.

    By default, this file is named Deployment<Year>-<Month>-<Day>_<Hours>-<Minutes>-<Seconds>_GMT.xml.

  2. On the Management Server, open a PowerShell window and enter the following command:

    cd "C:\Program Files\Unisys\EnterpriseMgmt\
     EcoAPI\discovery\bin"

  3. Enter the following command to execute the deploy project script:

    .\Deploy-Project.ps1 -xml <directory\project file name>.xml

    Where <directory\project file name> is the full path to the project file that you want to deploy. For example, C:\Windows\temp\Deployment2017-2-17_22-25-20_GMT.xml.

  4. When prompted, reenter the user name and enter the password for the Enterprise Manager Portal Administrator, Cryptographic Administrator, or Service Administration user.

  5. Wait while the script executes. You can monitor the progress of the script execution from one of the following:

    • The PowerShell console

    • The log file, which is located in the directory where you saved the project using the project name. By default, this file is named Deployment<Year>-<Month>-<Day>_<Hours>-<Minutes>-<Seconds>_GMT.log.

    When the execution of the PowerShell script is successful, the following message appears:

    Deploy Project <name of the project file> successfully completed.

    Note: If there are errors and the execution of the script is not successful, refer to the log file and retry the deployment if necessary with corrected input.

    The configuration is visible in Enterprise Manager.

  6. If you selected Create Objects Only from the Deploy tool, do the following.

    Note: If you selected Create and Deploy, the Stealth(aware) Model has been provisioned to the network, and you are ready to install the Stealth endpoint software on any system that you want to use Stealth for secure communications. See Installing Stealth Endpoint Software.

    1. Sign in to Enterprise Manager.

    2. View the created objects in Enterprise Manager.

       Caution

      You should use Enterprise Manager to view objects and provision only; you must not make any changes to the objects using Enterprise Manager. If you make any changes using Enterprise Manager, those changes will not be reflected in Stealth(aware); this means that your configuration and Stealth(aware) map will not be synchronized and a new deployment from Stealth(aware) could disable Stealth communications that you configured using Enterprise Manager.

      Therefore, if you see any objects that you want to change while reviewing the configuration in Enterprise Manager, return to Stealth(aware) to make the changes, and then select the Deploy tool again.

    3. Do either of the following to provision the environment:

      • Return to Stealth(aware), select the Deploy tool, and select the Create and Deploy option.

      • In Enterprise Manager, use the standard method to provision. See the Unisys Stealth Solution Information Center for more information.

Stealth(aware) Overview

Stealth(aware) helps you to visualize the network traffic and the Nodes in your computing environment. (Nodes are any devices or user identities included in your network.) To use Stealth(aware), you install the Stealth(aware) software on one system in your environment, and you install the Stealth Discovery Agent on selected Windows and Linux systems. Installing this software is not disruptive and can be performed at any time.

The Stealth Discovery Agent collects network traffic data that enables you to identify the Nodes in your environment and their communication paths.

Note: The Stealth Discovery Agent only captures 5-tuple values (source IPv4 address and port number, destination IPv4 address and port number, and the protocol in use).

You are then directed to import the network traffic data into Stealth(aware), and this is used to build a Stealth(aware) Model, which is a visual representation of your network. Based on the affinity level you select, you can see traffic flows and Nodes that have connections to one another in Stealth(aware).

Based on the network traffic data, Nodes are classified based on their role in the environment using a known set of characteristics. For example, Nodes can be identified as Database Servers, Web Servers, Application Servers, or Workstations based on the known differences in the way those applications communicate. After the network topology map is created, you can interact with the map to refine the model. Stealth(aware) can display up to 4,000 nodes and 8,000 traffic flow channels.

Stealth(aware) uses the Internet Assigned Numbers Authority (IANA) Allocation Guidelines for TCP and UDP port number assignments for classification. If your environment does not use the standard IANA port assignments, you have the flexibility to manually reclassify the Nodes based on their function in your environment. For example, you should reclassify your Database Server Nodes if they do not use the standard IANA port assignments for Database Servers.

When the map has been configured for your environment, you can click the Deploy button. Clicking Deploy translates the Stealth(aware) Model and creates the appropriate components to be imported into Stealth Enterprise Manager, including Stealth Communities of Interest (COIs), roles, users, groups, and filters. It creates the endpoint software packages that you can install on your Nodes to Stealth enable your environment. This ensures that each Stealth endpoint (Node running Stealth endpoint software) communicates securely with other endpoints as determined by the Stealth(aware) Model you create using Stealth(aware).

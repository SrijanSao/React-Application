Initial Discovery Guidelines

You should use an existing Network Map to identify optimal capture points that can be used to acquire network traffic information in packet capture (pcap or cap) format. The Network Map should include the following information about the datacenter:

  • Application characteristics and security requirements

  • IP addresses (identifying which IP addresses are private and non-routable, and which are public but are used in the data center)

    Private IP address ranges include:

    • 10.0.0.0/8 (10.0.0.0 - 10.255.255.255)

    • 172.16.0.0/12 (172.16.0.0 - 172.31.255.255)

    • 192.168.0.0/16 (192.168.0.0 192.168.255.255)

  • Ports

  • Host names

  • Roles

  • Existing segmentation (firewalls, VLANs, switch infrastructure, and network segments)

Optimal capture points are the systems that make up the center of activity for your solution or datacenter. These might include database servers or the key components and tiers of the application stack.

Note: It is recommended that you limit the size of each file you import to less than 30 MB to minimize the amount of time required to import data to Stealth(aware). You can either limit the time you spend capturing data, or you can divide one large file into multiple smaller files using a program like Wireshark editcap.

Configuring the AuthService.config File

The AuthService.config file is used to configure the Authorization Service running on the Management Server. For Stealth(aware), you must configure two AuthGroups as described in this topic. One AuthGroup is used to authorize the Management Server, and the second AuthGroup is used to authorize all other endpoints in the environment.

Do the following to configure the AuthService.config file for Stealth(aware):

  1. From the Start menu, enter Notepad in the Search box.

  2. Right-click Notepad, and select Run as administrator.

  3. If you are asked if you want to allow Notepad to make changes to this computer, click Yes.

  4. In Notepad, select the File menu, and then click Open.

  5. On the Open dialog box, change the file type from Text Documents (*.txt) to All Files.

  6. Browse to the C:\Program Files\Unisys\Stealth Solution\USS-AuthServer directory and open Sample(aware)AuthService.config.

    The Sample(aware)AuthService.config file includes two AuthGroups that are configured specifically for Stealth(aware).

     Caution

    Be sure to open the Sample(aware)AuthService.config file rather than the SampleAuthService.config file.

    The SampleAuthService.config file applies to Stealth(core) and includes additional AuthGroups and Authorization Schemes that are not supported in Stealth(aware).

  7. Open a new Notepad file using the Run as administrator option.

  8. Copy the first line in the sample file (<AuthService version="3.x"> into the new Notepad file.

    From the end of the sample file, copy the <AuthGroup> lines and the </AuthService> line into the new Notepad file.

    The new file should look like the following:

    <AuthService version="3.2">
  <AuthGroup name="Aware-Administration">
    <Url>http://192.168.36.20:10200/</Url>
    <AdminUrl>http://127.0.0.1:10210/</AdminUrl>
    <AuthenticationScheme>IntegratedWindowsAuthentication
       </AuthenticationScheme>
    <CertAge>1825</CertAge>
  </AuthGroup>

  <AuthGroup name="Aware-Project">
    <Url>http://192.168.36.20:10300/</Url>
    <AdminUrl>http://127.0.0.1:10310/</AdminUrl>
    <AuthenticationScheme>IntegratedWindowsAuthentication
       </AuthenticationScheme>
    <CertAge>1825</CertAge>
</AuthGroup>

</AuthService>

  9. For each AuthGroup, do the following in the <Url> line:

    • If the Management Server uses SSL, update the http:// value to https://.

    • Change the example IP address (192.168.36.20) to the static IPv4 address of your Management Server.

    Note: Do not change the port numbers. Do not change the <AuthGroup name>, <AdminURL>, or <AuthenticationScheme>.

  10. Optionally, for each AuthGroup, in the <CertAge> line, leave the default value or enter a new age (in days) for the certificate for the AuthGroup. (The certificate for the AuthGroup is created when you restart the Authorization Service as directed at the end of this procedure.)

    Note: The default value of 1825 days (or five years) is a standard for some self-signed certificates. Enter a value that is appropriate for your environment and security policies.

  11. From the File menu, select Save As in the new Notepad file.

  12. In the Save As dialog box, set the Save as type to All Files, enter the file name AuthService.config, and then click Save.

  13. If you are prompted for encoding, accept the default.

  14. Close the AuthService.config file, and close Notepad.

  15. Using Windows Explorer, browse to C:\Program Files\Unisys\Stealth Solution\USS-AuthServer, and verify that the AuthService.config file is located in this directory and that it does not have an extension of .txt.

  16. Open the file using Notepad, verify that your changes were saved, and then close Notepad.

  17. From the Start menu, enter services.msc in the Search box to access the Services console.

  18. Restart the Unisys Stealth Authorization Service so that the Authorization Service uses the AuthService.config file you saved.

Configuring Server Mode Using a User Name

Any Windows server that runs Stealth software should be configured to run with Server Mode enabled so that the server is automatically Stealth-enabled whenever it reboots. In general, any unattended system should be configured to run with Server Mode enabled. This includes the Management Server as well as Stealth-enabled AD servers, DNS servers, DHCP servers, NTP servers, Syslog servers, database servers, and web servers. (Your Stealth network might not include all of these types of servers.) Perform this procedure after you install the Stealth endpoint software on the server to configure the account that is used by the Unisys Stealth Logon service.

Note: If you configured silent installation, and if you entered the command to assign the service user credentials to the Unisys Stealth Logon service, skip this procedure. (Server mode is already configured.)

To configure Server Mode using a user name, perform the following procedure.

Do the following:

  1. Log in with a user name that has the appropriate authority to modify services, typically an administrative user name.

  2. Access Services. To do so, from the Start menu, enter services.msc in the Search box to access the Services.

  3. Locate the Unisys Stealth Logon service in the list of installed services.

    Note: Be sure to locate the Unisys Stealth Logon service (rather than the Unisys Stealth PreLogon service or the Unisys Stealth Protocol service).

  4. Right-click the Unisys Stealth Logon service and select Properties.

  5. On the Properties dialog box, select the Log On tab, and then select the This account option.

    Note: The user account you set to run in Server Mode must be in the administrators group on the local computer. If this is a domain\user name, the domain user must be added to the local computer administrators group.

  6. Enter the user name and password for the Stealth network. (These are the user credentials that determine the Stealth role and COIs when Stealth is enabled on the server.)

    When the user name is set in the account properties and a domain user account is being associated with the service, the administrator must use NT4 format (DOMAIN\username). The default UPN format (username@www.domain.com) cannot be used because of the problem described in the Microsoft Knowledge Base article 955641. For more information, go to http://support.microsoft.com/kb/955641.

    Note: By default, when using the Browse/Check Names functionality in the Services MMC Snap-in, domain accounts are returned in UPN format. This means that you must change the format of the account name to NT4 style explicitly. This also means that you must know the NetBIOS name of the domain (as the UPN format uses the fully qualified domain name). If you do not know the NetBIOS name of the domain, you should contact your network administrator.

  7. Click OK to close the dialog box.

  8. Click OK when you receive a warning that the changes do not take effect until you restart the service.

  9. Close the Services window.

  10. Reboot the server.

    Note: If you see a dialog box that requests Stealth credentials, you can click Cancel and continue with the reboot.

Using the StealthConfiguration Utility to Create Required Configurations

 Caution

In this procedure, you must spell and capitalize all commands exactly as written.

Therefore, you might want to do the following:

  1. Copy-and-paste these commands from this document into a text editor.

  2. Update the variables for your environment.

  3. Remove the line breaks.

  4. Paste the resulting command into the command prompt window by right-clicking and selecting Paste.

The StealthConfiguration Utility is a command line utility that runs on the Management Server. Based on the values you enter, it does the following:

  • Creates an Administrative configuration that is used to configure the Management Server

  • Creates an endpoint configuration that is used to configure all other endpoints in the environment so that they can communicate with the Management Server and be authorized

  • Creates a Stealth endpoint package that must be installed on the Management Server

  • Creates Stealth endpoint packages that must be installed on all of the other systems in your environment that you want to Stealth-enable so that they can communicate with one another securely using Stealth

To use the StealthConfiguration Utility do the following:

  1. Ensure that you are logged on to the Management Server as a local administrator user.

  2. Open a command prompt using the Run as administrator option.

  3. Enter the following command:

    cd "C:\Program Files\Unisys\EnterpriseMgmt\Stealth Manager"

  4. If you need to create any clear text filters, do the following. For example, if the Management Server needs to communicate with an Active Directory server that will not be Stealth-enabled, you can create a clear text filter, or if the endpoints need to communicate with a Domain Name System (DNS) server that will not be Stealth-enabled, you can create a clear text filter.

    You can use one clear text filter for the Management Server and use a different clear text filter for all other endpoints, or you can use one clear text filter that can be shared by the Management Server and by all endpoints.

    Note: Each clear text filter you create must have a unique name. You can create as many clear text filters as you choose, but you can only reference one clear text filter from the Management Server configuration and one clear text filter from the endpoint configuration.

    Each clear text filter can include one or more of the following, in a comma-separated list:

    • IP address – Enter one or more IPv4 addresses

    • IP address range – Enter one or more IPv4 address ranges, separated by a hyphen or using CIDR notation. For example, enter 192.0.2.8-192.0.2.16 or enter 192.0.2.0/24.

    • IP address with specific ports or protocols – Enter one or more IPv4 addresses or address ranges with port or protocol filters, using the syntax <IP address>:<protocol from 0 to 254> or <IP address>:<UDP or TCP>[<wild card or local port>:<wild card or remote port>]. For example, enter 198.51.100.3:20 or 198.51.100.3:TCP[*:53].

    To create one or more clear text filters, do the following:

    1. Enter the following command:

      StealthConfiguration.exe -v
     ClearTextFilterName="<Filter Name>" 
     ClearTextFilter="<IP address or addresses>"

      Notes:

      • Ensure that you spell and capitalize these commands exactly as written.

      • The -v option enables verbose mode and provides additional feedback on the results of the command. You should see feedback as the filter is created.

    2. Wait a brief time while the command is processed and the filter is created. When the processing is complete, you receive the command prompt.

    3. If you need to create additional clear text filters, reenter the command using a unique filter name and an updated IP address or list of IP addresses.

    For example, if you want to configure one filter set to communicate with a DNS server that will be used by both the Management Server and the endpoints, you could enter the following command:

    StealthConfiguration.exe 
     ClearTextFilterName="DNS" ClearTextFilter="192.0.2.8"

    In another example, if you want to configure one filter set for an Active Directory server for the Management Server, and configure a second filter set for the endpoints to communicate with the Active Directory server and with DNS servers on an IP address range using TCP port 53, you could enter the following command:

    StealthConfiguration.exe 
     ClearTextFilterName="MS_AD" ClearTextFilter="192.0.2.10"

StealthConfiguration.exe 
     ClearTextFilterName="EP_DNS" 
     ClearTextFilter=
          "192.0.2.10,198.51.100.1-198.51.100.5:TCP[*:53]"

  5. To create the configurations for your Management Server and endpoints, enter the following command:

    StealthConfiguration.exe -v
     StealthConfigName="<Admin Configuration Name>"
     ClearTextFilterName="<Optional Management Server filter>"
     AdminType=AwareAdmin
     AdminUserName="<Admin User Name>"
     AdminAuthorizationPort=10200 
     UserAuthorizationPort=10300 
     IPAddress="<Management Server IP Address>" 
     ProjectName="<Endpoint Configuration Name>"
     ProjectClearTextFilterName=
          "<Optional endpoint filter>"

    Notes:

    • Ensure that you spell and capitalize these commands exactly as written.

    • The -v option enables verbose mode and provides additional feedback on the results of the command. You should see feedback as the configurations are created.

    The following is a description of each of these values:

    Value

    Required/ Optional

    Explanation

    StealthConfigName=“<Admin Configuration Name>”

    Required

    Enter a name for the Administrative configuration used by the Management Server. For example, enter StealthConfigName=“AwareAdmin”.

    ClearTextFilterName=“<Optional Management Server filter>”

    Optional

    If you created a clear text filter you want to use in the Administrative configuration for the Management Server, enter the name of a filter you configured in the previous step. For example, enter ClearTextFilterName=“MS_AD”.

    AdminType=AwareAdmin

    Required

    This is not a variable. Enter this value exactly as specified.

    AdminUserName=“<Admin User Name>”

    Required

    Enter the name of the Administrator user that you configured the Stealth Enterprise Manager services to run under. For example, enter “AdminUserName=“EMAdmin”

    AdminAuthorizationPort=10200

    		Required

    This is not a variable. Enter this value exactly as specified.

    UserAuthorizationPort=10300

    		Required

    This is not a variable. Enter this value exactly as specified.

    IPAddress=“<Management Server IP Address>”

    Required

    Enter the static IPv4 address of the Management Server. For example, enter IPAddress=“198.51.100.10”.

    ProjectName=“<Endpoint Configuration Name>”

    Required

    Enter a name for the for the endpoint configuration. For example, enter ProjectName=”AwareEndpoints”.

    ProjectClearTextFilterName= ”<Optional endpoint filter>”

    Optional

    If you created a clear text filter you want to use in the endpoint configuration, enter the name of a filter you configured in the previous step. For example, if you created a filter named EP_DNS, enter ProjectClearTextFilterName=”EP_DNS”.

    For example, enter the following: 

    StealthConfiguration.exe -v
     StealthConfigName="AwareAdmin" 
     ClearTextFilterName="MS_AD" 
     AdminType=AwareAdmin 
     AdminUserName="EMAdmin"
     AdminAuthorizationPort=10200 
     UserAuthorizationPort=10300 
     IPAddress="198.51.100.10" 
     ProjectName="AwareEndpoints"
     ProjectClearTextFilterName="EP_DNS"

  6. Wait a brief time while the configurations and endpoint packages are created.

    When the processing is complete, you receive the command prompt.

  7. To verify that the configurations and endpoint packages were created, do the following:

    1. Using Windows Explorer, navigate to C:\Stealth Files\SoftwareInstalls\Packages.

    2. Verify that the following six endpoint packages were created:

      • <Admin Configuration Name>AwareWinPkg64-SAO.exe

      • <Endpoint Configuration Name>AwareLinuxPkg-SAO.sh

      • <Endpoint Configuration Name>AwareWinPkg32-SAO.exe

      • <Endpoint Configuration Name>AwareWinPkg32-COD.exe

      • <Endpoint Configuration Name>AwareWinPkg64-SAO.exe

      • <Endpoint Configuration Name>AwareWinPkg64-COD.exe

If you do not see all of these endpoint packages, or if you want to delete your configurations and begin again, see Updating Configurations Using the StealthConfiguration Utility.

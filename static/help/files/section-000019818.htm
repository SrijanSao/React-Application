Silent Installation Procedure

If you want to silently install the Stealth endpoint software on Windows endpoints, enter the following command using a command prompt window on the endpoint, or programmatically distribute and install this software using a software distribution system, such as System Center Configuration Manager (SCCM).

Notes:

  • By default, when silent installation is run, the executable file terminates without waiting for the installation to complete. To wait for the installation to complete before terminating the endpoint package executable file, preface the command with Windows command start /wait.

    Alternatively, to run the endpoint package executable file with silent options from a Windows script (such as PowerShell or VBScript), use the command shell Run and specify the wait option.

  • Depending on the system settings and how the installer is run, silently installing the Stealth endpoint software might result in the appearance of a Windows User Account Control (UAC) pop-up message. For information on UAC and options to manage it for silent installations, see the following Microsoft TechNet article:

    https://technet.microsoft.com/en-us/library/cc709691

Silent Installation for a Server Using a User Name and Password for Server Mode

For a server on which you want to configure Server Mode using a user name and password, enter the following command from an Administrator command prompt to install Stealth silently. This command configures Server Mode by assigning the service user credentials to the Unisys Stealth Logon service.

<directory\endpoint package name>.exe -ap "/qn
     /serviceuser=<domain>\<user> /servicepw=<password>"

For example, enter C:\temp\Winpackage.exe -ap "/qn /serviceuser=Admin.loc\AdminUser /servicepw=resU*nimdA"

Notes:

  • The user must belong to the local Administrators group.

  • The user can be a local user or a domain user. If you do not enter a domain, the command assumes that the user is part of the local domain (.\user). If you do enter a domain, you can enter either the fully qualified domain name (FQDN) or the domain short name; for example, /serviceuser=[<domain>\]<user>.

  • If you use a domain user, either the Active Directory server must be Stealth-enabled and the endpoint package must be configured as Stealth Always On, or the endpoint package must include a clear text filter to the Active Directory server. (You cannot use a Stealth On Demand endpoint package if the Active Directory server is Stealth-enabled, because Stealth is not enabled on the endpoints at the time that the Stealth Logon Service starts, when authentication with the Stealth-enabled Active Directory server is required.)

Additional Options

Additionally, you can add the following options to the command:

  • /nomsu – Add this option to skip the check for required Windows updates for Windows 7 and Windows Server 2008 R2, if you know that all updates are installed on the endpoint.

  • /nodiag – Add this option to skip the diagnostic collection that occurs, in the event of a failed installation and roll back.

If there are any problems encountered during installation on the endpoints, they are listed in the log file C:\Stealth\StealthEP_Install.log and in the Windows Application Event log.

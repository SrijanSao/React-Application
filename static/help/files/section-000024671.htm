Configuration Requirements for Integrating with Stealth(core)

In order to use Stealth(aware), your environment must meet the following requirements for integrating with Stealth(core):

  • You must have a new environment in which to deploy Stealth.

    Using Stealth(aware) in an existing Stealth environment is not supported in this release, and there is no upgrade path that includes use of Stealth(aware). That is, there should be no existing Stealth Management Server, standalone Authorization Servers, or Stealth endpoints in the network in which you want to configure Stealth(aware); you are directed to create these new components later in this book.

    However, if you have an existing Stealth environment and want to use Stealth(aware), you could configure a separate network in which to deploy a new Stealth environment (for example, a new test environment).

  • You must use the Management Server to authorize all endpoints. (Standalone Authorization Servers are not supported in this release.)

  • Your Management Server must have a static IPv4 address.

  • Your environment must use IPv4 addressing.

  • You must use all default certificates created by Enterprise Manager, including the default Signing certificate (and its associated Trusted Root certificate) and the default Authorization Service certificates.

  • Your endpoint users must be included in Active Directory groups in order to be authorized. This means that the Management Server must belong to an Active Directory domain. In addition, the Active Directory server must either be Stealth-enabled, or the Management Server and endpoints must include clear text filters to the Active Directory server.

    (For endpoint authorization, local users and Active Directory users are not supported for this release; only Active Directory groups are supported.)

  • You must authorize all endpoints using Integrated Windows Authentication. (LDAP and certificate-based authorization are not supported in this release.)

  • You must have a datacenter configuration that does not require use of the Secure Remote Access (SRA) Gateway or the Secure Virtual Gateway.

If your environment does not meet these requirements, you must use Enterprise Manager to configure your Stealth environment, including creating and provisioning configurations, COIs, roles, users, and groups. See the Unisys Stealth Solution Information Center for more information.

Certificate Requirements

Enterprise Manager uses a signing certificate to protect the integrity of the settings.xml and crypto.xml files, which are included in the endpoint software package. The endpoint uses the trust chain (that is, the corresponding root and intermediate certificates) to validate the signing certificate, which is then used to validate these XML files before enabling Stealth.

For Stealth(aware), you must use the signing certificate and root certificate that are automatically generated when Enterprise Manager is initialized. When directed, you must import the trusted root certificate for the signing certificate onto each endpoint that you want to Stealth-enable.

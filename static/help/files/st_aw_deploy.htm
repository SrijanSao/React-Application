Deploying Solutions to Enterprise Manager

Note: If you set the Affinity Level to Off, you cannot use this tool.

Using Stealth(aware), do the following to deploy your Solutions to Enterprise Manager.

Best Practices:

  • Before you deploy your Solutions, ensure that you manually updated the Solution name. Updating the Solution name ensures that auto modeling will not make any changes to those Solutions after you deploy them.

    For example, you could append _D (for deployed) for each Solution name you intend to deploy.

  • In general, you should deploy all Solutions in your project each time you deploy. You can choose to not deploy a particular Solution only if that Solution is not ready for deployment. In that case, you can continue configuring the Solution and deploy it at a later time. Note the following:

    • If a Solution includes Stealth-enabled Profiles, all of the required objects are created in Enterprise Manager (including configurations, COIs, roles, and filters).

    • If a Solution does not include Stealth-enabled Profiles, the required clear text filters to enable communication to those Nodes are created in Enterprise Manager.

    • If a Solution does not include at least one Profile, it is ignored.

    • If the Solution has been previously deployed (even if you made no changes to the Solution) you must select it to be deployed, or it will be deleted from Enterprise Manager.

To deploy one or more Solutions using the Deploy tool, do the following:

  1. On the map, click the Deploy tool.

             

    The map turns gray and a panel appears that prompts you to select options and then click Deploy.

  2. Under Select an enforcement level for deployment, optionally select a new enforcement level from the list. This setting provides a guideline for the filters that are created by Stealth(aware); however, filtering is also dependent on Channel policies and whether or not the connection is in clear text.

    Note: You initially configured this setting in Updating the Project Name and Details (Project Details Panel). If you change this value on the Deploy panel, it will be updated in the Project Details panel as well, and future deployments will use this updated setting by default.

    Optionally update the value and select one of the following:

    • Enforce IP Addresses and Services – Filters are generated to enforce IP addresses and both local and remote ports.

    • Enforce IP Addresses – Filters are generated to enforce local ports on the providing services and IP addresses on the consuming profiles.

    • Enforce Services – Filters are generated to enforce local ports on the services.

    • Don’t enforce either IP Addresses or Services – Filters are only generated when they are required.

  3. Under Select the Solutions you want to deploy, select the check box for each Solution that you want to deploy, or click Clear/Select All to clear the list or select all check boxes.

    Note: You can choose to not deploy a particular Solution only if that Solution is not ready for deployment. In that case, you can continue configuring the Solution and deploy it at a later time.However, if the Solution has been previously deployed (even if you made no changes to the Solution) you must select it to be deployed, or the associated objects created in Enterprise Manager will be deleted and the associated Nodes will not be able to use Stealth to communicate.

  4. Select one of the following options:

    • Create Objects Only

      If you select Create Objects Only, an XML file is created that enables Stealth objects to be created in Enterprise Manager, but the Solutions are not automatically deployed. You can view the created objects in Enterprise Manager, and then use Enterprise Manager to provision the Stealth environment.

    • Create and Deploy

      If you select Create and Deploy, an XML file is created that enables Stealth objects to be created in Enterprise Manager, and the Solutions are then automatically deployed. (There is no opportunity to review the created objects in Enterprise Manager before the Stealth environment is provisioned.) It is a best practice to select this option, because it simplifies deployment and can help to avoid errors in using Enterprise Manager.

    For either option you select, a window appears that prompts you to save the project as an XML file.

    The default name for each file is Deployment<Year>-<Month>-<Day>_<Hours>-<Minutes>-<Seconds>_GMT in Greenwich Mean Time (GMT).

  5. Browse to any location and save the project.

    The Deploy tool closes.

  6. If Stealth(aware) is not running on the Management Server, proceed to Copying and Deploying the Project File to Enterprise Manager.

    If Stealth(aware) is running on the Management Server, do the following:

    1. From the PowerShell window that appears, enter the user name and password when prompted.

      Note: The credentials that you enter must be for a member of the Enterprise Manager Portal Administrator, Cryptographic Administrator, or Service Administration group.

    2. Wait while the script executes. You can monitor the progress of the script execution from one of the following:

      • The PowerShell console

      • The log file, which is located in the directory where you saved the project using the project name. By default, this file is named Deployment<Year>-<Month>-<Day>_<Hours>-<Minutes>-<Seconds>_GMT.log.

      After the PowerShell script is successfully executed, the following message appears:

      Deploy Project <name of the project file> successfully completed.

      Note: If there are errors and the execution of the script is not successful, refer to the log file and retry the deployment if necessary with corrected input.

      The configuration is visible in Enterprise Manager.

  7. If you selected Create Objects Only from the Deploy tool, do the following.

    Note: If you selected Create and Deploy, the Stealth(aware) Model has been provisioned to the network, and you are ready to install the Stealth endpoint software on any system that you want to use Stealth for secure communications. See Installing Stealth Endpoint Software.

    1. Sign in to Enterprise Manager.

    2. View the created objects in Enterprise Manager.

       Caution

      You should use Enterprise Manager to view objects and provision only; you must not make any changes to the objects using Enterprise Manager. If you make any changes using Enterprise Manager, those changes will not be reflected in Stealth(aware); this means that your configuration and Stealth(aware) map will not be synchronized and a new deployment from Stealth(aware) could disable Stealth communications that you configured using Enterprise Manager.

      Therefore, if you see any objects that you want to change while reviewing the configuration in Enterprise Manager, return to Stealth(aware) to make the changes, and then select the Deploy tool again.

    3. Do either of the following to provision the environment:

      • Return to Stealth(aware), select the Deploy tool, and select the Create and Deploy option.

      • In Enterprise Manager, use the standard method to provision. See the Unisys Stealth Solution Information Center for more information.

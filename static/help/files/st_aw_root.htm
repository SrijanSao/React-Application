Defining Roots for Solutions (Create Solutions Tool)

The Create Solutions tool enables you to select the Profiles to be used as the basis—or root—of Solutions. A new Solution is created for every root selected.

For example, if you choose Automated—which automatically selects all Profiles classified as Database Servers to be used as roots—and if your Model includes seven Database Server Profiles, then seven Solutions are created.

Note: If you set the Affinity Level to Off, you cannot use this tool.

Classifying and Organizing Profiles and Nodes

Before you begin using the Create Solutions tool, you should ensure that your Profiles and Nodes have been classified appropriately by Stealth(aware). Classification plays an important part in modeling. Nodes are classified based on their role in the environment using a known set of characteristics. For example, Nodes can be identified as Database Servers, Web Servers, Application Servers, or Workstations based on the known differences in the way applications running on Nodes communicate. Profiles are classified based on the Nodes that they include.

Stealth(aware) uses the IANA Allocation Guidelines for TCP and UDP port number assignments for classification. If your environment does not use the standard IANA port assignments, you have the flexibility to manually reclassify the Nodes based on their function in your environment. For example, you should reclassify your Database Server Nodes if they do not use the standard IANA port assignments for Database Servers.

You can change the classification assigned by Stealth(aware) by doing the following:

  1. Select a Profile whose classification you want to change, or that includes a Node whose classification you want to change.

  2. In the Profile Details panel, do one of the following:

    • If you want to change the Profile classification, select a new classification from the list at the top of the Profile Details panel.

      See Modifying Profile Details (Profile Details Panel) for more information.

    • If you want to change a Node’s classification, select that Node from the Nodes list, and then update the Node classification using the list at the top of the Node Details panel.

      See Modifying Node Details (Node Details Panel) for more information.

If all of the Nodes in a Profile have the same classification, the Profile will have the same classification as the Nodes. If Nodes in the same Profile have different classifications, the Profile uses the Mixed classification. However, you can manually update the Profile classification as required.

In addition, from the Profile Details panel, you can move Nodes into and out of Profiles. Each discovered IP address is displayed as an individual Node. (For systems that include multiple NICs with different IP addresses, each individual NIC is displayed as its own Node.) Therefore, you might want to combine these Nodes into one Profile, if they are not already grouped.

Using the Create Solutions Tool

Modeling uses the consumer and provider relationships of the Profiles to create Solutions. To model your environment using the Create Solutions tool, do the following:

  1. Click the Create Solutions tool.

             

    The Create Solutions panel appears.

  2. From the Create Solutions panel, under Rooting method, select one of the following:

    • Automated (recommended) – Automatically creates Solutions based on all Profiles that are classified as Database Servers.

       Caution

      If you select Automated but no Profiles have been classified as Database Servers, then no Solutions will be created.

    • Manual – Enables you to manually select Profiles that you want to use as the root of your Solutions, either because no Database Servers have been identified or because you want to select Profiles manually.

  3. Under Profile Roots (used to create Solutions), do the following based on whether you selected Automated or Manual:

    • If you selected Automated, review the list of Database Server Profiles, which are automatically used as the root of your Solutions.

      The Database Server Profiles are also highlighted on the Stealth(aware) map.

    • If you selected Manual, select the Profiles you want to use from the map.

      The Profiles you select are also highlighted on the Stealth(aware) map.

      Note: If you select Manual, it is a best practice to select Profiles that are classified as either Database Servers, Application Servers, or Web Servers as roots in order to generate stable Solutions. Selecting these types of servers as roots increases the likelihood that the Solutions will not require manual changes before deployment.

      You can remove a Profile from the list by clicking the X associated with the Profile, or you can remove all Profiles by clicking Clear All Profiles.

  4. If you selected Automated, under Automatically create new Solutions, determine the setting you want to use, as follows.

    Note: This setting does not appear if you selected Manual.

    • If you set this value to Off, when new traffic data is imported, no changes are made to existing Solutions, and new Solutions are not automatically created. (However, new Nodes are affinitized into existing Profiles and Solutions on the map. See Setting the Affinity Level (Set Affinity Level Tool) for more information on affinitization.)

      Note: The Automatically create new Solutions function is Off by default.

    • If you set this value to On and import new traffic data, any new Profiles classified as Database Server are automatically used to create Solutions, and any profiles that Stealth(aware) has not moved into existing Solutions are automatically moved into the Internet or Intranet Solutions. (You do not see Profiles on the map before they are added to a Solution, unless they are classified as User Profiles.)

    Note: If you set this value to On, any empty Solutions in the model are deleted. To avoid this, add at least one Profile to the empty Solution.In addition, any Profiles and Solutions that you manually modified or created are ignored. For example, if you turn Automatically create new Solutions on, it will not do the following:

    • Move a Profile into a Solution that you modified or created

    • Move a Profile out of a Solution that you modified or created

    • Delete a Solution that you modified or created (if it includes any Profiles)

  5. Click Proceed.

Profiles that are related to the Solutions you selected as roots are sorted into Solutions based on the roots you selected. Any remaining Profiles are automatically moved into the Internet or Intranet Solutions.

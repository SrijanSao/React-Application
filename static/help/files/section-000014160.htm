Importing the Trust Chain for the Endpoint Package Signing Certificate on Windows Endpoints

A signing certificate is used to protect the integrity of the settings.xml and crypto.xml files, which are included in the endpoint software package. The XML files in the endpoint software package are signed, and that signature is verified by the signing certificate.

The trusted root certificate for the default signing certificate, which is created by Enterprise Manager, is located on the Management Server in the C:\Stealth Files\Certificates directory, and it is named Unisys Stealth Root. Before proceeding, copy the Unisys Stealth Root trusted root certificate from this directory and import it to the Local Computer/Trusted Root Certification Authorities store on each endpoint.

Note: The signing certificate is automatically included in the endpoint package, which means that the signing certificate does not have to be imported on any endpoints.

 Caution

If you do not import the required certificates, the endpoint cannot be authorized and the Stealth service cannot be started.

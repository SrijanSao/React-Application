Setting the Affinity Level (Set Affinity Level Tool)

After you import traffic data, you can use the Set Affinity Level tool to set the threshold for grouping similar Nodes into Profiles.

Using the Set Affinity Level tool, select the affinity level that most closely aligns with your Network Map. The default value is Medium.

Note: If Nodes have very little in common, changing the Affinity Level will have little or no effect on the map.

To change the Affinity Level, do the following:

  1. Click the Set Affinity Level tool.

             

    The Set Affinity Level panel appears.

  2. From the Set Affinity Level panel, select one of the following:

    • Low – Profiles are created with a larger number of Nodes that are less closely related to one another and that are providing and consuming some of the same services.

    • Medium – This is the default value. A number of Profiles between the Low and High value are created, and they are providing and consuming many of the same services.

    • High – Profiles are created with a smaller number of Nodes that are more closely related to one another and that are providing and consuming almost all of the same services.

    • Off – All Profiles are removed from the map, and you see the Nodes and their underlying relationships to one another.

      Note: If you set the Affinity Level to Off, you cannot use any of the other tools (for example, Import Traffic Data or Deploy).

  3. When prompted, click Yes, Change.

    The Processing message appears, and then closes when processing is complete.

Note: If you change the Affinity Level, any empty Solutions in the model are deleted. To avoid this, add at least one Profile to the empty Solution.

Affinitization ignores Profiles and Solutions that you manually modified or created. You can manually create a Profile or Solution using the Add Objects tool, as described in Manually Creating Objects (Add Objects Tool). You can manually modify a Profile or Solution by updating the name of the Profile or Solution from the Details panel. Additionally, you can manually modify a Solution by dragging and dropping a Profile into that Solution.

There is no way to see if an existing Profile or Solution has been manually modified or created. You should make a note of any Profiles or Solutions that you manually modify or create. Stealth(aware) does not inform you of Profiles and Solutions that have been ignored by Affinitization or auto modeling.

When you import new traffic data, new Nodes are affinitized into existing Profiles and Solutions on the map.

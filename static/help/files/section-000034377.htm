Understanding Time to Import Network Traffic Data to Stealth(aware)

When importing network traffic data files into Stealth(aware), the best indicator of the required processing time is the number of Nodes in a network. You should plan to allow adequate time for Stealth(aware) to process the network traffic data and for the information to appear on the Stealth(aware) map so that the processing workflow is not disrupted.

Note that other factors—such as the hardware capabilities of the system running Stealth(aware) and other properties of a given network—can affect the processing time, so these times should be considered approximate.

For reference, the following table lists tested processing times for environments with various numbers of Nodes and associated Profiles and Channels as well as different network traffic data file sizes.

 Caution

The following processing times were tested using a physical system that met the requirements of Stealth(aware), as described in Stealth(aware) System Requirements. If you use a virtual machine to run Stealth(aware)—which is not recommended—it could take far longer to import network traffic data files. This is because virtualization server load, network performance, and other virtualization factors can have a significant impact on virtual machine processing time.

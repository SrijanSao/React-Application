Installing the Management Server Software

You install the Management Server software using the Unisys Stealth Solution Release 3.x ISO file. The ISO file includes two separate software installation packages: one for installing the Management Server software and one for installing the standalone Authorization Server software.

Note: Standalone Authorization Servers are not supported for Stealth(aware). Do not install a standalone Authorization Server in your environment.

The installer checks that the prerequisite Management Server software (and correct software versions) are installed. If any prerequisite software is not yet installed, or if the wrong software level is installed, the installer launches various windows to guide you through that software installation. Once all of the prerequisite software packages are installed, the Enterprise Manager software is installed and launched.

Note: Two logs are created in the C:\Stealth directory during the installation process: EMLiferayTomcatInstall.log and EMPortalInstall.log. If you experience any issues with the installation, you can refer to these logs.

Complete the following steps to install the software on your Management Server:

  1. Ensure that you met the Management Server requirements in Management Server Requirements.

     Caution

    Do not proceed with the installation until you meet the Management Server requirements. If you do not meet these requirements, the installation will fail.

  2. Log on to the Management Server using the user name and password that the Stealth Enterprise Manager services should run under.

    Note: It is a best practice to use a local user rather than a domain user. You must be sure that the user is included in the Local Administrator Group for the Management Server and is assigned user rights to Log on as a service.

  3. Ensure that the ISO file you downloaded is mounted to a local drive or extracted to a local drive.

    Note: You cannot install the Management Server software from a remote or shared drive.

  4. If it does not run automatically, double-click the Autorun.exe file from the ISO file to launch the installer.

    Note: Be sure to select the Autorun Application (.exe), rather than the Autorun Setup Information file.

  5. Click Yes if you are prompted to allow the installer to make changes to the computer.

    The Unisys Stealth Solution Software CD screen appears.

  6. Click the Install button next to Stealth Management Server software.

    The open-source license for the Enterprise Manager Liferay Portal and Tomcat Server components appear.

  7. Read the license terms that apply to Liferay Portal, Tomcat Server, as well as to MySQL Server, and then click Accept Terms.

    The installer checks for prerequisite software that is not already installed and displays the first software installer. (Subsequent installers start automatically.)

  8. If .NET Framework 4.6.1 is not already installed, click Upgrade to upgrade to the required version of .NET Framework.

    Note: If .NET Framework 4.6.1 is already installed (and you see a prompt to install Microsoft Visual C++ 2015 x86 Redistributable), skip to the next step.

    Do the following to install .NET Framework if the installation screens for this software appear:

    1. Wait while the software is extracted.

    2. On the .NET Framework 4.6.1 Setup screen, read the license terms, select the I have read and accept the license terms check box, and then click Install.

    3. Wait while the software is installed, which can take several minutes.

    4. When the Installation Is Complete screen appears, click Finish.

      Note: If you are prompted to reboot your system, do so. Then, log back on using the user name and password that you want the Stealth Enterprise Manager services to run under, and relaunch the installation process by following the steps at the beginning of this procedure.

  9. Wait while the Microsoft Visual C++ installation is launched.

    Wait while the following software packages are automatically installed (if they are not already installed):

    • Microsoft Visual C++ 2015 x86 Redistributable

    • Microsoft Visual C++ 2015 x64 Redistributable

    • Microsoft Visual C++ 2010 x86 Redistributable

    • Microsoft Visual C++ 2010 x64 Redistributable

    The Installer launches the Java Setup wizard, if Java version 8 is not already installed.

  10. If Java version 8 is not already installed, do the following.

    Note: If Java version 8 is already installed, skip to the next step.

    1. On the Java Setup – Welcome page, click Install.

      Note: Do not change the destination folder.

    2. When you see the “You have successfully installed Java” screen, click Close.

  11. When prompted, enter a password for the MySQL root account, confirm the password, and then click Next.

    Make a note of this password and record it in a safe location; it is required if you update the Management Server software.

    Note: If this is a new MySQL installation, you can use the same password that you used to log on to the Management Server, or you can enter a unique password for MySQL.

    The MySQL Installer launches. Wait while the software is installed.

  12. When prompted, enter the password and confirm the password for the user name that you used to log on to the Management Server, and then click Next.

    Note: As described in Management Server Requirements, this user name must be included in the Administrator Group for the Management Server and must be assigned user rights to Log on as a service. This is the user name that the Enterprise Manager services will run under.

  13. Wait until the Stealth Enterprise Manager prompt appears and asks: “Do you want to restore a previously saved Stealth configuration to the database for Stealth Enterprise Management?”

  14. Click No (nothing to restore) for a new installation.

  15. Wait for the Enterprise Manager software to be installed.

  16. When you are prompted to configure the user account to be created for Tomcat Server, do the following:

    1. Accept the default user name Tomcatuser, or enter a new user name in the Account name box.

      If you enter a new user name, it must be unique in your environment.

    2. Enter and confirm a password for Tomcat Server.

      The password must be at least eight characters long, and it must include at least three of the following: uppercase letters, lowercase letters, numbers, and symbols. In addition, the user name cannot be included as part of the password.

      You can use a password that you used previously, or you can enter a unique password.

    3. Accept the default Tomcat Server port number of 29080, or enter a new port number.

      This is the port that will be used to access Enterprise Manager.

    4. Click Next

    5. Wait while the Liferay Portal and Tomcat Server software is installed. It can take more than five minutes to install the Liferay Portal and Tomcat Server software.

      Note: The Tomcat Server user is configured with a password that never expires. Do not change this setting. If this password expires, you will be unable to log into Enterprise Manager.

  17. When you are prompted to configure the user account to be created for the Stealth Ecosystem API (EcoAPI), do the following:

    1. Accept the default user name ecoapiUser, or enter a new user name in the Account name box.

      If you enter a new user name, it must be unique in your environment.

    2. Enter and confirm a password for the EcoAPI.

      The password must be at least eight characters long, and it must include at least three of the following: uppercase letters, lowercase letters, numbers, and symbols. In addition, the user name cannot be included as part of the password.

      You can use a password that you used previously, or you can enter a unique password.

    3. Accept the default EcoAPI port number of 8448, or enter a new port number.

      This is the port that will be used to access the EcoAPI.

    4. Click Next

    5. Wait while the associated software is installed.

  18. After the EcoAPI is installed, wait while the base Enterprise Manager software is installed and Stealth Enterprise Management Portal software is installed. This software installation can take several additional minutes to complete.

    During the installation process, two icons are created on the Management Server desktop that enable you to

    • Uninstall the Unisys Stealth Enterprise Manager software (the icon is named “Uninstall Unisys Stealth Enterprise Management”)

    • Launch Enterprise Manager (the icon is named “Unisys Stealth Enterprise Management Portal”)

    In addition, a shortcut is added to the Start menu that enables you to easily collect diagnostic information for the Management Server. To access the shortcut, from the Start menu, enter Collect Diagnostics in the Search box, right-click, and select Run as administrator. See the Unisys Stealth Solution Information Center for more information on collecting diagnostics.

    When the installation is complete, on the Unisys Stealth Solution Software CD screen, you see “Status: Stealth Enterprise Management Portal [<version>] installed. Portal deployment successful.”

    Note: If the initial Enterprise Manager deployment fails, the installer automatically retries the deployment. Wait for Enterprise Manager to deploy successfully; the retry process can take several minutes to complete.

  19. If you want to use your default browser, click Launch, and then wait while Enterprise Manager is configured and launched.

    If you want to use another browser, click Close to close the installer, and then enter the following URL in a browser window:

    https://<computer name>:<port number (by default, 29080)>/

    Note: The Management Server computer name must be able to be resolved by a Domain Name System (DNS) server or other name resolution mechanism. If the Management Server is in a domain, the shortcut created to launch Enterprise Manager uses the fully qualified domain name (FQDN) of the Management Server, and therefore the FQDN must be resolvable. If the computer name is not resolvable, you might have to manually update the shortcut to use the computer name (without the domain) or localhost.

  20. After launching Enterprise Manager, enter the portal administrator user name and password, and then click Sign In.

    For a new installation, the default user name is portaladmin and the default password is AdminPortal1#.

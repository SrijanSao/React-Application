Modeling Tools

The vertical bar of icons in the upper-left of the map are called the Modeling Tools. These include the following:

  • Import Traffic Data tool

  • Set Affinity Level tool

  • Add Objects tool

  • Create Solutions tool

  • Deploy tool

See Using Stealth(aware) to Create a Model for more information.

Stealth(aware) Model Objects

The Stealth(aware) Model can include the following objects:

  • Solution – A Solution is an application stack; for example, an order processing, ERP, or HR application.

    The Stealth(aware) Affinitization and auto modeling process identifies those devices and user identities (Nodes) in the network that have frequent communications with each other, and are therefore likely to represent a single application stack. A Solution is comprised of Profiles and their connecting Channels.

  • Profile – A Profile is a container for devices that perform a similar function; for example, Web Servers, Application Servers, and Database Servers. Profiles are defined on the Stealth(aware) Map through the automated Affinitization of the network traffic flows recorded during Stealth Discovery and using the auto modeling process.

    Affinitization identifies those devices that include similar traffic flows to other devices, which enables Stealth(aware) to create Profiles that combine similar devices into a single object. During the process of creating Profiles, individual devices (for example, Web Servers) are also auto-classified using a heuristics analysis of the traffic data to and from each device.

    That is, affinitization helps simplify the modeling process by combining similar devices together and representing them as a single object on the map.

  • Channel – A Channel contains the set of individual device traffic flows either between Profiles (External Channels) or among members of the same Profile (Internal Channels).

    Each traffic flow is defined by the source and destination of the IPv4 address and port, by the direction of the traffic, as well as by the TCP or UDP communications protocol.

    Communication Policies can be set for Channels on individual Profiles or traffic flows while building the Stealth(aware) Model.

  • Node – A Node is any device or user identity in the computing environment.

    When Affinitization is set to Off, Nodes appear on the map. When Affinitization is set to Low, Medium, or High, Nodes with the same classification are affinitized into Profiles, with Channels drawn between each of the Profiles containing the Nodes.

    In this release, each discovered IP address is displayed as an individual Node. (For systems that include multiple NICs with different IP addresses, each individual NIC is displayed as its own Node.)

Note: If the name of a Solution, Profile, or Node is too long, it will be appear concatenated on the map.

Standard Solutions

The top of the map displays the following standard solutions that are automatically created and always displayed on the map. After the Auto Modeling process finishes, the following Solutions are populated:

  • Intranet Solution – A Solution that contains Profiles (and their Nodes) that were not added to an existing Solution during the Auto Modeling process. This includes Profiles that are classified as Workstations.

    The address space (that is, the range of IP addresses in the private range) is predefined and cannot be modified.

  • Infrastructure Solution – A Solution to which you add Profiles (and their Nodes) that were not added to an existing Solution during the Auto Modeling process. You should drag-and-drop Profiles for infrastructure servers (such as DNS, AD, Domain Controllers, and routers) that will never be Stealth-enabled into this Solution.

  • Enterprise Manager Solution – A Solution that contains the Profile for the Management Server. The Management Server Profile is the root of the Enterprise Manager Solution. If you classify a Profile as a Management Server, you can manually drag-and-drop this Profile into the Enterprise Manager Solution.

  • Internet Solution – A Solution that contains Profiles (and their Nodes) that were not added to an existing Solution during the Auto Modeling process. This Solution contains Profiles that will never be Stealth-enabled.

    The address space in the public range is predefined and cannot be modified.

Model Details

In a Stealth(aware) Model, a Profile Chain defines the relationship between service providers and service consumers. Each consumer of the network traffic data is located below the provider of that service. The Create Solutions tool is used to define Profiles as the basis, or root, to create a Solution. The default root is Database Profiles. Each vertical stack of Profile Chains begins at the Database Server (by default) and ends at the Mixed Profile (if it exists). In a Solution, Profiles are arranged based on the classification order as follows:

  • Database Server

  • Database Proxy

  • Application Server

  • Application Proxy

  • Web Server

  • Web Proxy

  • Admin Users

  • Admin Workstations

  • Users (non-admin)

  • Workstations (non-admin)

  • Mixed

Profiles include Nodes and Internal or External Channels that encompasses all members of the Profile.

Channels are the lines on the map that connect Profiles containing Nodes to one another. If Channels connect two Profiles that are Stealth-enabled, you can apply Stealth policies to flows within the Channel. Those Channels are a composite of all of the traffic flows that have been observed between those two objects.

Note the following:

  • Any Profiles that are classified as Workstations (even Workstations that were previously in the Profile Chains), but are not yet Stealth-enabled, are automatically moved into the Intranet Solution during the auto modeling process.

    This includes the Profiles that Stealth(aware) has classified as Workstations or that you have manually classified as Workstations.

  • Any Profiles that are classified as Users—whether classified by Stealth(aware) or manually classified by you—remain at the bottom of the model. These profiles can be manually Stealth-enabled.

    If you want to deploy these Profiles, you must manually drag-and-drop a new Solution onto the map using the Add Objects tool, and then move those Profiles into that Solution. It is recommended that you name that Solution “Stealth Users.”

  • Any profiles that are not modeled into Solutions are added to the Intranet or Internet Solutions, based on the IP addresses of the Nodes in the Profile. If the Profile contains Nodes that fall into both ranges of IP addresses (Internet and Intranet), the Profile are divided and moved into the appropriate solution.

Clicking on a Solution expands the Solution to display the Profiles and their connecting Channels within that Solution. Profiles that have a blue background are designated to be Stealth-enabled.

After using the Create Solutions tool, each new Solution contains the Profiles that were modeled into each Solution. The user can manually select the profiles used as roots, or the Stealth(aware) algorithms can automatically select them for the user.

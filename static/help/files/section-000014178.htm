Importing the Trust Chain for the Endpoint Package Signing Certificate on Linux Endpoints

A signing certificate is used to protect the integrity of the settings.xml and crypto.xml files, which are included in the endpoint software package. The XML files in the endpoint software package are signed, and that signature is verified by the signing certificate.

During the Linux endpoint installation processes, you are directed to do the following:

  • Obtain the trust chain for the signing certificate in PEM format.

    The trusted root certificate for the default signing certificate that is created by Enterprise Manager is located on the Management Server in the C:\Stealth Files\Certificates directory, and it is named Unisys Stealth Root. You can copy the Unisys Stealth Root trusted root certificate from this directory and import it on each endpoint.

  • If you do not already have one in your environment, set up the CA certificate store. The certificate store can be a certificate file that contains the contents of the trust chain certificates, or it can be another method that is specific to your environment.

  • Import the trust chain certificates into the certificate store.

  • Specify the location of the CA certificate store in the /etc/stealth/system.ini file.

Note: The signing certificate is automatically included in the endpoint package, which means that the signing certificate does not have to be imported on any endpoints.

See Linux Endpoint Initial Installation and Authorization Configuration for directions on installing the Stealth endpoint software, including importing the trust chain certificates to the CA certificate store, and specifying the location in the system.ini file.

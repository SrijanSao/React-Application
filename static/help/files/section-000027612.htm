Configuring a Windows File Share for the Discovery Agent Network Traffic Data Files

When you install the Discovery Agent on Nodes, you must determine which Windows file share the network traffic data files (in packet capture (pcap) format) should be sent to.

Note: In normal operations, you do not send these network traffic data files directly to the system running Stealth(aware), because that system might not be accessible to all Nodes. However, in a Proof-of-Concept environment, you could send these network traffic data files directly to the system running Stealth(aware).

The file share must be a Common Internet File System (CIFS) file share, which is a Windows native file-sharing protocol.

If you have not already done so, use the standard Windows method to configure a Windows file share on a system to which you want to send the network traffic data (pcap) files. This file share should be accessible to the following:

  • The system running Stealth(aware)

  • The Nodes running the Discovery Agent

Best Practice: For security, configure the file share to enable write-access for the Nodes that you want to run the Discovery Agent.

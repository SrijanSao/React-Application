Domain Participation

It is a requirement for the Management Server to be part of a domain; make sure that it is part of the appropriate domain before proceeding. Your endpoint users must be included in Active Directory groups in order to be authorized. This means that the Management Server must belong to an Active Directory domain. In addition, the Active Directory server must either be Stealth-enabled, or the Management Server and endpoints must include clear text filters to the Active Directory server. (For endpoint authorization, local users and Active Directory users are not supported for this release; only Active Directory groups are supported.)

 Caution

It is critical that you determine what resources are required for Stealth endpoints (including the Management Server) to function within the domain, and identify whether those resources should be Stealth-enabled or clear text. For example, if your Domain Controller, Active Directory, and Domain Name System (DNS) servers are not Stealth-enabled, you must assign a filter to roles so that the Stealth endpoints can communicate in clear text to those servers. If your Domain Controller, Active Directory, and DNS servers are Stealth-enabled, they must share a Service COI and a workgroup COI with the roles they authorize.

Information about creating filters or Stealth-enabling servers is available later in this document.

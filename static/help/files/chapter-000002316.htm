Installing Stealth Endpoint Software

When you run the StealthConfiguration Utility (as described in Using the StealthConfiguration Utility to Create Required Configurations), the required Stealth endpoint packages are created in the C:\Stealth Files\Software Installs\Packages directory on the Management Server. When you use the Deploy tool in Stealth(aware), you provision the Stealth Management Server so that it can authorize endpoints and assign them to the appropriate roles and COIs.

You use the following endpoint packages to install the Stealth software on the workstations and servers that you want to Stealth-enable:

  • <Endpoint Configuration Name>AwareWinPkg32-SAO.exe

    Windows 32-bit, Server Always On – Use this package to configure Windows 32-bit operating systems.

    This mode is generally used for servers or any unattended system in the environment that need to run Windows Server Mode. Stealth is enabled when the system boots and cannot be disabled using the Stealth Dashboard.

  • <Endpoint Configuration Name>AwareWinPkg32-COD.exe

    Windows 32-bit, Client On Demand – Use this package to configure Windows 32-bit operating systems.

    This mode is generally used for Windows workstations or any regularly attended Windows system in the environment. Stealth is disabled when the system boots. Stealth remains disabled until the user specifically enables it using the Stealth Dashboard.

  • <Endpoint Configuration Name>AwareWinPkg64-SAO.exe

    Windows 64-bit, Server Always On – Use this package to configure Windows 64-bit operating systems.

    This mode is generally used for servers or any unattended system in the environment that need to run Windows Server Mode. Stealth is enabled when the system boots and cannot be disabled using the Stealth Dashboard.

  • <Endpoint Configuration Name>AwareWinPkg64-COD.exe

    Windows 64-bit, Client On Demand – Use this package to configure Windows 64-bit operating systems. This mode is generally used for Windows workstations or any regularly attended Windows system in the environment. Stealth is disabled when the system boots. Stealth remains disabled until the user specifically enables it using the Stealth Dashboard.

  • <Endpoint Configuration Name>AwareLinuxPkg-SAO.sh

    Linux, Server Always On – Use this package to configure all Linux endpoints (32-bit or 64-bit). Stealth can be enabled or disabled using a command.

Copy these endpoint packages to the workstations and servers on which you want to install Stealth, and use the appropriate endpoint package based on the operating system type, architecture, and behavior.

Note: Do not install the <Admin Configuration Name>AwareWinPkg64-SAO.exe package on any system except the Management Server.

Best Practice: Before you begin the endpoint software installation, it is a best practice to do the following:

  • Ensure that the workstations and servers that you want to Stealth-enable have network connectivity and are on a network segment that would be able access the Management Server.

  • Ensure that there are no firewalls blocking Stealth communications. For details on configuring firewalls to enable Stealth communications, see the Unisys Stealth Solution Information Center.

Deleting the Endpoint Configuration and Creating a New Endpoint Configuration

If you want to delete the endpoint configuration and create a new one, do the following:

  1. Ensure that you are logged on to the Management Server as a local administrator.

  2. Open a command prompt using the Run as administrator option.

  3. Enter the following command to access the StealthConfiguration Utility directory:

    cd "C:\Program Files\Unisys\EnterpriseMgmt\Stealth Manager"

  4. Enter the following command to delete the endpoint configuration:

    StealthConfiguration.exe -v DeleteProject="<Endpoint Configuration Name>"

    Notes:

    • Ensure that you spell and capitalize this command exactly as written.

    • The -v option enables verbose mode and provides additional feedback on the results of the command. You should see feedback as the project is deleted.

    For example, enter StealthConfiguration.exe -v DeleteProject="AwareEndpoints".

  5. If you need to create any new clear text filters, do the following. For example, if the endpoints need to communicate with a Domain Name System (DNS) server that will not be Stealth-enabled, you can create a clear text filter.

    Note: Each clear text filter you create must have a unique name. You can create as many clear text filters as you choose, but you can only reference one clear text filter from the new endpoint configuration.

    Each clear text filter can include one or more of the following, in a comma-separated list:

    • IP address – Enter one or more IPv4 addresses

    • IPv4 address range – Enter one or more IPv4 address ranges, separated by hyphen or using CIDR notation. For example, enter 192.0.2.8-192.0.2.16 or enter 192.0.2.0/24.

    • IP address with specific ports or protocols – Enter one or more IPv4 addresses or address ranges with port or protocol filters, using the syntax <IP address>:<protocol from 0 to 254> or <IP address>:<UDP or TCP>[<wild card or local port>:<wild card or remote port>]. For example, enter 198.51.100.3:20 or 198.51.100.3:TCP[*:53].

    To create one or more clear text filters, do the following:

    1. Enter the following command:

      StealthConfiguration.exe -v
     ClearTextFilterName="<Filter Name>" 
     ClearTextFilter="<IP address or addresses>"

      Notes:

      • Ensure that you spell and capitalize these commands exactly as written.

      • The -v option enables verbose mode and provides additional feedback on the results of the command. You should see feedback as the filter is created.

    2. Wait a brief time while the command is processed and the filter is created. When the processing is complete, you receive the command prompt.

    3. If you need to create additional clear text filters, reenter the command using a unique filter name and an updated list of IP addresses.

    For example, if you want to configure one filter set to communicate with a DNS server that will be used by both the Management Server and the endpoints, you could enter the following command:

    StealthConfiguration.exe 
     ClearTextFilterName="DNS" ClearTextFilter="192.0.2.8"

    In another example, if you want to configure one filter set for an Active Directory server for the Management Server, and configure a second filter set for the endpoints to communicate with the Active Directory server and with DNS servers on an IP address range using TCP port 53, you could enter the following command:

    StealthConfiguration.exe 
     ClearTextFilterName="MS_AD" ClearTextFilter="192.0.2.10"

StealthConfiguration.exe 
     ClearTextFilterName="EP_DNS" 
     ClearTextFilter=
          "192.0.2.10,198.51.100.1-198.51.100.5:TCP[*:53]"

  6. To create a new endpoint configuration, enter the following command:

    StealthConfiguration.exe -v
     StealthConfigName="<Admin Configuration Name>"
     UserAuthorizationPort=10300 
     IPAddress="<Management Server IP Address>"
     ProjectName=”<Endpoint Configuration Name>”
     ProjectClearTextFilterName=
          "<Optional endpoint filter>"

    Notes:

    • Ensure that you spell and capitalize these commands exactly as written.

    • The -v option enables verbose mode and provides additional feedback on the results of the command. You should see feedback as the configurations are created.

    The following is a description of each of these values:

    Value

    Required/ Optional

    Explanation

    StealthConfigName=“<Admin Configuration Name>”

    Required

    Enter the name for the Administrative configuration used by the Management Server. This must be the name you entered in Using the StealthConfiguration Utility to Create Required Configurations. For example, enter StealthConfigName=“AwareAdmin”.

    UserAuthorizationPort=10300

    		Required

    This is not a variable. Enter this value exactly as specified.

    IPAddress=“<Management Server IP Address>”

    Required

    Enter the static IPv4 address of the Management Server. This must be the name you entered in Using the StealthConfiguration Utility to Create Required Configurations. For example, enter IPAddress=“198.51.100.10”.

    ProjectName=“<Endpoint Configuration Name>”

    Required

    Enter a new name for the for the endpoint configuration. For example, enter ProjectName=”AwareEndpoints1”.

    ProjectClearTextFilterName= ”<Optional endpoint filter>”

    Optional

    If you created a clear text filter you want to use on the endpoint configuration, enter the name of a filter you configured in the previous step. For example, if you created a filter named EP_DNS, enter ProjectClearTextFilterName=”EP_DNS”.

    If you receive any errors that updated endpoint packages could not be created, ignore those errors.

  7. Perform the steps in Deploying the Stealth(aware) Model to redeploy from Stealth(aware).

    Note: When you update the endpoint configuration, you do not have to install a new version of Stealth endpoint software. When you deploy from Stealth(aware), you configuration is reprovisioned, and the Stealth endpoints automatically use the updated endpoint configuration information.

Troubleshooting Stealth(aware) Errors

If Stealth(aware) encounters any errors, use the information in this topic to understand and troubleshoot those errors.

Table 8. Stealth(aware) Errors

Error

Explanation

Troubleshooting Steps

Error 001: There was an unexpected error and Stealth(aware) is not usable.

Due to a critical failure, Stealth(aware) is not usable.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. Uninstall and reinstall Stealth(aware).

  4. See Restoring a Previous Version of Stealth(aware) for information on restoring your previous environment.

Error 101: There was an unsupported upgrade or downgrade attempt.

There was an attempt to upgrade Stealth(aware) from a version that is not supported, or there was an attempt to downgrade Stealth(aware).

See Restoring a Previous Version of Stealth(aware) for information on restoring your previous environment.

Error 102: There was a template mismatch during an upgrade.

During a supported upgrade, an unexpected error condition occurred, and the upgrade process was compromised.

  1. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  2. See Restoring a Previous Version of Stealth(aware) for information on restoring your previous environment.

  3. Obtain a new version of the ISO file, and then attempt the upgrade again.

Error 103: The database could not be successfully migrated during an upgrade.

During a supported upgrade, an unexpected error condition occurred, and the database migration could not be completed.

  1. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  2. See Restoring a Previous Version of Stealth(aware) for information on restoring your previous environment.

Error 201: The database could not be opened and might be corrupted.

Due to an attempted manual change to the database or a critical system failure, the database could not be opened.

See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

Error 301: A rooting error occurred.

There was an unexpected error when running the Create Solutions tool, if you used the Default (All Databases) option.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. See Backing Up the Stealth(aware) Database for information on restoring your database from a previously backed up version.

Error 302: An auto-modeling error occurred.

There was an unexpected error when running the Create Solutions tool, if you used the Manually Select Profiles option.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. See Backing Up the Stealth(aware) Database for information on restoring your database from a previously backed up version.

Error 303: An affinitization error occurred.

There was an unexpected error when running the Set Affinity Level tool.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. See Backing Up the Stealth(aware) Database for information on restoring your database from a previously backed up version.

Error 304: There was an affinitization error after processing the following pcap file: <file name>

After the traffic data file was imported, the software attempted to affinitize the new data based on the value in the Set Affinity Level tool. An unexpected error then occurred.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. See Backing Up the Stealth(aware) Database for information on restoring your database from a previously backed up version.

Error 305: There was an affinitization error after processing the pcap files imported using Live Feed. 

After the traffic data file was imported using the Live Feed option, the software attempted to affinitize the new data based on the value in the Set Affinity Level tool. An unexpected error then occurred.

  1. Close Stealth(aware) and then attempt to relaunch it.

  2. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys.

  3. See Backing Up the Stealth(aware) Database for information on restoring your database from a previously backed up version.

Error 401: There was an error reading the following pcap file: <file name>

An error occurred when reading the traffic data file. The file is likely corrupted.

  1. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys, including the file.

  2. Attempt to import a different traffic data file.

Error 402: There was a error reading the following pcap file imported using Live Feed: <file name>

An error occurred when reading the traffic data file using Live Feed. The file is likely corrupted.

  1. See Collecting Diagnostics for Stealth(aware) for information on sending diagnostics to Unisys, including the file.

  2. Access the directory you configured in Configuring a Windows File Share for the Discovery Agent Network Traffic Data Files, and remove the corrupted file from the share.

  3. Turn Live Feed on (it was automatically disabled when the error occurred).

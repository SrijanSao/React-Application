Installing the Discovery Agent on Nodes Running Windows

You can install the Discovery Agent on a Node running one of the Windows operating systems listed in Discovery Agent Supported Operating Systems.

As part the installation process, the following software is installed on the Windows Node, if it is not already installed:

  • Microsoft Visual C++ 2015 Redistributable x86 or x64, depending on the system architecture

  • Win10Pcap

Do the following to install the Discovery Agent on Nodes running Windows:

  1. Save the Discovery Agent executable file in a location that is accessible from your Windows Nodes. This file is named Stealth_Discovery_Installer_Release_<x64 or x86> and is included with the Stealth(aware) ISO file in the discovery\windows folder.

  2. From Windows Explorer, right-click the Discovery Agent executable file for your system type, and then select Run as administrator.

  3. If you are prompted to allow the program to make changes to your computer, click Yes.

  4. On the Unisys Stealth(aware) Auto Discovery window, click Install.

  5. If it is not already installed, wait while Microsoft Visual C++ 2015 Redistributable is installed.

  6. If Win10Pcap is not already installed, wait while it is automatically installed.

    When the installation is complete, the Unisys Stealth(aware) Auto Discovery Setup Wizard launches.

  7. Click Next.

  8. It is recommended that you retain the destination folder default location, and then click Next.

  9. Enter the appropriate values on the Discovery Share Settings page, as follows:

    • Remote Share – Enter the network address of the share you configured in Configuring a Windows File Share for the Discovery Agent Network Traffic Data Files.

    • Remote Share User – Enter a user name that has privileges to write to this share.

    • Remote Share Password – Enter the password for the user.

      Note: This password cannot be updated unless you reinstall the Discovery Agent software. Therefore, you should enter credentials for a user whose password does not expire.

    Click Next.

  10. Enter the appropriate values on the Discovery Collection Settings page, as follows:

    • When you are asked how often (in seconds) you would like to start a capture, enter a new value or retain the default value of 840 seconds (14 minutes).

    • When you are asked how long (in seconds) you would like each capture to last, enter a new value or retain the default value of 65 seconds (1 minute, 5 seconds).

    • When you are asked how often (in seconds) you would like to move completed capture files to the remote directory, enter a new value or retain the default value of 1800 seconds (30 minutes).

    Note: The value for The length of time (in seconds) that each capture sessions lasts should be equal to or less than the value for The time interval (in seconds) that a new capture is started. If these values are the same, this means that data is continuously captured.

  11. Click Next.

  12. Click Install.

  13. When the installation is complete, click Finish.

  14. When you see a message that reads “Installation Successfully Completed,” click Close.

 Caution

If the remote share, user name, and password are not correct, the installation will fail. If the installation fails, do the following:

  1. Perform this procedure again, and verify that the Discovery Share Settings page includes the correct information.

  2. For more information, see the Discovery.txt log file, which is located in C:\ProgramData\Unisys\Stealth.

The process of discovering data begins automatically.

If you want to stop the discovery process, do the following to stop the Unisys Stealth Discovery service:

  1. From the Start menu, enter Services in the Search box, right-click Services, and select Run as administrator.

  2. Locate the Unisys Stealth Discovery service in the list of Services.

  3. Right-click the Unisys Stealth Discovery service and select Stop.

Modifying Profile Details (Profile Details Panel)

Use the Profile Details panel to view (and modify, if necessary) details for a specific Profile, including the classification of the Profile, the Profile name, and default policies, as described in the following procedure.

Viewing and Modifying a Profile

To view the Profile Details panel and modify details of a Profile, do the following:

  1. Select a Profile directly from the map, or select a Profile by first selecting a Solution and then selecting a Profile from the expanded Solution or from the Solution Details panel.

    A Profile has a blue background if it is Stealth-enabled and a white background if it is not Stealth-enabled.

    The Profile Details panel appears.

  2. If necessary, click the classification icon at the top of the panel and select a new classification from the list.

    The classification of the Profile is based on the classification of all of the Nodes included in the Profile.

    The following table provides a description of each classification.

    Notes:

    • If the classification icon is a question mark (?), the Profile is unclassified, either because it could not be automatically classified or because you created it manually using the Add Objects tool. If you select a classification for an unclassified Profile, you cannot change the Profile back to unclassified.

    • Stealth(aware) does not automatically classify any Profiles as Admin Users. You can manually classify Profiles as Admin Users, as required.

    Note: This indicates that Nodes of different classifications are included in the same Profile.


    Note: The Stealth Management Server, Stealth Standalone Authorization Server, and Stealth Service Mode classifications also appear in this list. Ignore these classifications; they are not supported in this release.

  3. If necessary, in the Profile Name field, modify the name of the Profile. This name can be a maximum length of 45 characters and must meet the requirements in Supported Characters and Length Restrictions in Enterprise Manager and Stealth(aware).

  4. If necessary, click the policy icon to the right of the Profile Name field and select a new policy from the list to change the Inter-Profile Policy, which sets the default for all traffic flows that connect to this Profile.

    Note: The Inter-Profile Policy is only visible when the Stealth Enabled setting is set to On.

    The following table provides a description of each policy:

    Table 4. Policies

    IconDescription

    Stealth Allow – Allow Stealth traffic.

    Clear Text Allow – Allow clear text traffic.

    Block Clear Text and Stealth – Block clear text and Stealth traffic.


  5. In the Group Name field, specify the name of the Active Directory group that the Profile is associated with. (That is the name of the Active Directory group in which the users of the member Nodes are included.) This name can be a maximum length of 45 characters and must meet the requirements that are described in Supported Characters and Length Restrictions in Enterprise Manager and Stealth(aware).

    Entering information in this field is required so that your endpoints can be authorized. This becomes the Active Directory Group that identifies your Stealth endpoint users.

    If you have not already done so, use your Active Directory server to create a group and then configure users to be members of that Active Directory group. For example, if you are configuring a Database Server Profile, and if your Database Server users are located in an Active Directory group named DB_Server_Admins, then you would enter DB_Server_Admins in the Group Name field in Stealth(aware).

    Note: If the Profile is part of a Solution, you see the Solution field under the Group Name field. If necessary, you can click the name of the parent Solution to access the Solution Details panel. See Modifying Solution Details (Solution Details Panel) for more information. (If the Profile is not part of a Solution, you do not see the Solution field.)

  6. Use the Stealth Enabled switch to determine if the Profile should be Stealth-enabled by selecting one of the following:

    • On – The Profile will be Stealth-enabled. The background color of the Profile is blue.

    • Off – The Profile will not be Stealth-enabled. The background color of the Profile is white.

     Caution

    Be very careful when changing the Stealth Enabled setting. If you turn this setting to Off and deploy the associated Solution, the Profile and its Nodes will not be Stealth enabled. Even if you install the Stealth endpoint software on an associated Node, that Node will not be able to use Stealth communications if Stealth is not enabled on its Profile.

    Notes:

    • If you set the Stealth Enabled switch to On, you should ensure that the Policy is set to Stealth Allow.

    • By default, the Profile is set to Stealth-enabled or not Stealth-enabled based on whether the Solution switch Stealth Enabled Added Profiles is set to On or Off. However, if you modify the Profile before adding it to a Solution, the Profile will not use the default Stealth value of the Solution.

  7. If necessary, from the Nodes list, do the following:

    • To remove a Node from the Profile, drag-and-drop the Node onto the map. The Node is moved onto the map into a new Profile, where it is the only member.

    • To view or update details for a member Node using the Node Details panel, click a Node from the list. Nodes are listed by IP address, host name, or name (depending on what values are defined).

  8. If the Profile includes an Internal Channel, you can review the flows and if necessary, update the policy on each flow, as follows:

    • If the Profile is Stealth enabled, set a Stealth policy, either Stealth Allow or Block Clear Text and Stealth.

      Note: Although you can set Clear Text Allow as an option, this is not recommended for traffic flows that connect two Stealth-enabled Profiles.

    • If the Profile is not Stealth enabled, the Clear Text Allow policy is set by default and cannot be changed, but no policy is visible.

  9. Click the X in the upper-right corner to close the Profile Details panel.

    Note: When you make a change to the Profile Details panel, the change is automatically saved.

Deleting a Profile

To delete a Profile, do the following:

  1. Select the Menu icon in the upper-right corner of the Profile Details panel.

             

  2. Click Delete Profile.

  3. When prompted, click Yes, Delete.

If the Profile you are deleting includes any Nodes, those Nodes are each returned to the map in a new Profile.

Viewing External Channels for the Profile

To view details of the External Channels for this Profile, click the line going from the selected Profile to a different Profile or a Solution.

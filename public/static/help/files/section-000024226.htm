Optionally Configuring SSL to Work with the Authorization Service

If the Management Server is running SSL, and if you want to use SSL with the Authorization Service, do the following:

  1. Obtain an Authorization Service SSL certificate for your server.

    Note: This certificate common name must match the fully qualified domain name, IP address, or host name that you entered for the Url in Configuring the AuthService.config File, which endpoints use to connect to the Authorization Service.

    You can do any of the following:

    • Purchase certificate-management software and use the provided documentation to create and install the certificate.

    • Create and install your own self-signed certificate using a program such as MakeCert.

    • Contact a certificate authority, purchase a certificate, and then install it.

  2. Install the Authorization Service SSL certificate in the Local Computer Personal key store.

  3. Ensure that the trust chain for the certificate (including the trusted root certificate and any intermediate certificates) is installed in the Local Computer Trusted Root Certification Authorities key store.

  4. Do the following to locate the certificate thumbprint for the SSL certificate on the MMC snap-in window:

    1. Access the MMC utility, and run the Certificates snap-in.

      To run the Certificates snap-in, select Add/Remove Snap-in from the File menu. Click Add, select Certificates and then click Add.

      On the Certificates snap-in dialog box, verify that Computer account is selected, and then click Next.

      Select Local computer, and then click Finish. Click Close on the Add Standalone Snap-in dialog box. Click OK on the Add/Remove Snap-in dialog box.

    2. In the Certificates MMC snap-in window, in the left pane, expand Certificates (Local Computer), expand Personal, and then click Certificates.

    3. In the right pane, double-click the SSL certificate.

    4. Verify that the trust chain is valid. To do so, double-click the certificate, and then click the Certification Path tab. You should not see any errors stating that the trust chain could not be verified, and the Certificate status should read OK.

    5. On the Certificate dialog box, click the Details tab, and scroll down to view the thumbprint.

    6. Select the thumbprint value and copy it to the clipboard (press Ctrl+C).

  5. Enter the following at a command prompt that is run as administrator.

    Notes:

    • In the following command, the <AuthGroup URL> and <port number> should be the same values you entered for the first AuthGroup in the AuthService.config file. However, note that in the AuthService.config file, you can enter either the server IP address or fully qualified domain name; for this command, you must enter the IP address.

    • Right-click and select Paste to paste all of the alphanumeric characters for the <certificate thumbprint>, and then delete the spaces.

    • In the following command, the appid value is not a variable; enter it exactly as specified.

    Enter

    netsh http add sslcert ipport=<AuthGroup URL>:<port number>
     certhash=<certificate thumbprint>
     appid={350ad21e-58b2-4f19-aa0a-5e49e1d64de2}

    For example, enter

    netsh http add sslcert ipport=192.168.233.34:9000 
     certhash=beda74645d906bd7e639e8c1ge99d774264cg1g9
     appid={350ad21e-58b2-4f19-aa0a-5e49e1d64de2}

  6. Verify your SSL certificate by entering the following command: 

    netsh http show sslcert

  7. For each additional AuthGroup that uses SSL, repeat the previous steps using the appropriate AuthGroup <Url> and <port number> in the netsh command.

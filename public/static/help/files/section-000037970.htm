What's New in Release 3.2.104?

The following major changes are new for release 3.2.104:

  • A new topic provides approximate processing times required to import network traffic data files to Stealth(aware). See Understanding Time to Import Network Traffic Data to Stealth(aware).

  • New settings on the Project Details panel enable you to set DNS Resolution and to select a default Enforcement level for the Project. See Updating the Project Name and Details (Project Details Panel).

  • A new topic provides guidelines for how to complete initial discovery in your environment. See Initial Discovery Guidelines.

  • There are new best practices for installing the Stealth Discovery Agent in your environment. See Understanding Where to Install the Stealth Discovery Agent.

  • A new topic provides guidelines on how long the Stealth Discovery Agent should run. See Understanding How Long the Stealth Discovery Agent Should Run.

  • Updates to the Set Affinity Level tool enable you to set the Affinity to Off, which removes all Profiles from the map and displays the Nodes and their underlying relationships to one another. See Setting the Affinity Level (Set Affinity Level Tool).

  • The appearance and behavior of the Create Solutions tool has changed. The Create Solutions tool now combines the Root Selection function for Profile Chains with the Automatically create new Solutions function. See Defining Roots for Solutions (Create Solutions Tool).

  • The Solution, Profile, and Node Details panels have all been updated to display additional settings. In addition, you can now delete Solutions and Profiles from their Details panels. See Modifying Solution Details (Solution Details Panel), Modifying Profile Details (Profile Details Panel), and Modifying Node Details (Node Details Panel).

  • There are now additional ways to move Nodes between Profiles and to move Profiles between Solutions. See Moving Nodes Between Profiles and Moving Profiles Between Solutions.

  • A new option has been added to the Deploy tool that enables Stealth objects to be created in Enterprise Manager, without the Solutions being automatically deployed. This enables you to view the created objects in Enterprise Manager, and then use Enterprise Manager to provision the Stealth environment. See Deploying Solutions to Enterprise Manager.

  • A new topic provides recommendations on regularly backing up the Stealth(aware) database. See Backing Up the Stealth(aware) Database.

  • The procedure to upgrade Stealth(aware) has been updated. See Upgrading Stealth(Aware).

  • A new topic provides information about creating a new database file if you are modeling a different network segment in the same environment or if you are modeling a different environment. See Modeling New Environments .

  • A new topic provides information on troubleshooting Stealth(aware) errors. See Troubleshooting Stealth(aware) Errors.

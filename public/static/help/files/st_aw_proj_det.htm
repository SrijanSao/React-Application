Updating the Project Name and Details (Project Details Panel)

Use the Project Details panel to update the Project name and details, as follows:

  1. In the upper-left corner of the map, click the Project name, for example, Untitled 1.

    The Project Details panel appears.

  2. In the Project Name field, enter a new name for the project.

    The default name is Untitled x, where x is a generated number.

    The new name can include any characters that are supported by Microsoft and can be a maximum length of 45 characters.

    Note: Enter a unique name for each new project you create.

  3. For DNS Resolution, if you want the DNS servers that are configured on the system running Stealth(aware) to attempt to resolve the host name associated with each Node, leave the default value as On. This process runs continuously and automatically to attempt to resolve host names as new Nodes are added to Stealth(aware) and to confirm that existing host names are correct. Host names are displayed on the Node Details panel and on the Profile Details panel (only if a custom Node Name has not been entered).

    If you do not want the DNS Servers to attempt to resolve host names, select Off.

  4. In the Description field, enter a description of your project that you can use to help you identify it (up to 255 characters).

  5. Under Enforcement, leave the default value or select a new enforcement level from the list.

    Note: Alternatively, you can set this value when you deploy the project, as described in Deploying the Stealth(aware) Model.

    This setting provides a guideline for the filters that are created by Stealth(aware); however, filtering is also dependent on Channel policies and whether or not the connection uses clear text.

    Select one of the following:

    • Enforce IP Addresses and Services – Filters are generated to enforce IP addresses and both local and remote ports.

    • Enforce IP Addresses – Filters are generated to enforce local ports on the providing services and IP addresses on the consuming Profiles.

    • Enforce Services – Filters are generated to enforce local ports on the services.

    • Don’t enforce either IP Addresses or Services – Filters are only generated when they are required.

  6. Click Save Project.

    The project is saved on the local system. The Project Details panel closes, and the updated project name appears on the upper-left corner of the map.

Note that the Project Details pane also includes the following:

  • Created – The date and time that the Project was created in local system time.

  • Last Updated – The date and time that the Project was last updated.

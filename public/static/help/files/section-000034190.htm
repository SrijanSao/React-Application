Configuration Requirements

The Management Server has the following configuration requirements:

  • Create a new user or configure an existing user as an Administrator user for the Stealth Enterprise Manager services to run under.

    Best Practice: It is a best practice to use a local user rather than a domain user.

    This user must be included in the Local Administrator Group for the Management Server and assigned user rights to Log on as a service.

    To configure (or verify) the user account you want to use for this purpose has the appropriate user rights, do the following:

    1. Log on to the system you want to use as the Management Server using any administrator credentials.

    2. Ensure that you have created a user that you want Stealth Enterprise Manager services to run under. If you have not yet done so, use the standard Windows method to add a user.

      Note: Do not use the user name “Administrator” or “portaladministrator” for this user; those names are reserved for use by internal Enterprise Manager functions.

    3. If your Management Server is part of a domain, ensure that the user you want the Stealth Enterprise Management services to run under is configured as a user in that domain.

    4. Access Windows Local Security Policy.

      To do so, from the Start menu, enter Local Security Policy in the Search box.

    5. In the Local Security Policy window, expand Local Policies.

    6. Select User Rights Assignment.

    7. Right-click Log on as a service, and then select Properties.

    8. In the Log on as a service Properties dialog box, select Add User or Group to add the user that you want the Stealth Enterprise Manager services to run under.

    9. In the dialog box that appears, ensure that the From this location box displays either the domain that the user belongs to or the local computer name, depending on whether you want the Enterprise Manager services to run under a domain user name or a local user name. (It is a best practice to use a local user rather than a domain user.)

      If needed, click Locations and update the location.

    10. In the Enter the object names to select box, enter the user name, and then click Check Names.

    11. Click OK to close the open dialog boxes.

    12. Ensure that the local or domain user you added is also included in the Local Administrator Group for the Management Server.

      (Do not close the Local Security Policy window.)

  • Ensure that endpoint users you plan to authorize from the Management Server can Access this computer from the network.

    Do the following:

    1. On the Local Security Policy window, select User Rights Assignment.

    2. Click Access this computer from the network, and ensure that the endpoint users you plan to authorize from the Management Server have access. For example, ensure that this field reads Everyone.

    3. Close the Local Policy Security window.

  • Ensure that only one network adapter is enabled.

    The Stealth software cannot be installed successfully if more than one network adapter is enabled. Do the following to disable additional network adapters:

    1. From Control Panel, select Network and Internet.

    2. Select Network and Sharing Center.

    3. In the left pane, click Change adapter settings.

    4. Disable all network adapters—except the one you want to use for Stealth communications—by right-clicking the network adapter and selecting Disable.

      Note: You can reenable additional network adapters after you complete the installation.

    5. Close the Network and Sharing Center window.

  • Enable HTTP Activation and Microsoft Message Queueing Server (MSMQ).

    Do the following to enable HTTP Activation and MSMQ (or verify that it is enabled):

    • For Windows Server 2012 R2, do the following:

      1. Access the Server Manager Dashboard, select the Manage tab, and then select Add Roles and Features.

      2. If you see the Before you begin screen, click Next.

      3. If you see the Select installation type screen, select Role-based or feature-based installation, and then click Next.

      4. In the left menu, select Features.

      5. In the center pane, expand .NET Framework 4.x Features.

      6. Expand WCF Services, and then select HTTP Activation (if it is not already installed).

        If this software is already installed, skip to the next step.

        If this software is not already installed, in the Add Roles and Features dialog box, click Add Features.

      7. In the Add Roles and Features wizard, in the center pane, expand Message Queuing, expand Message Queueing Services, and then select Message Queuing Server (if it is not already installed).

        If this software is already installed, skip to the next step. If this software is not already installed, click Next, and then follow the prompts to install the software.

        Note: If you are prompted to close the wizard after installing the software, perform the first three steps to reopen the Add Roles and Features Wizard, Server Roles page.

      8. In the left pane, select Server Roles (if it is not already selected).

      9. In the center pane, under Web Server (IIS), under Web Server, select Common HTTP Features (if it is not already selected).

      10. Click Next.

      11. If you selected any software to install, select Confirmation in the left pane, and then proceed to the next step.

        If you did not select any software to install, close the window and proceed to the next procedure.

      12. On the Confirmation screen, click Install to install any additional software.

      13. When the installation is complete, click Close.

    Note: If you previously installed the Stealth software, you must access Server Manager and restart the Unisys Stealth Authorization Service to complete the HTTP activation.

  • Set the Microsoft Message Queueing Server (MSMQ) service to restart if it encounters a failure.

    Do the following:

    1. Access Services. To do so, from the Start menu, enter services.msc in the Search box.

    2. Locate the Message Queuing service in the list of services, right-click, and then select Properties.

    3. On the Properties dialog box, click the Recovery tab.

    4. In the First failure, Second failure, and Subsequent failures lists, select Restart the Service.

    5. Click OK to close the Properties dialog box.

    6. Close the Services window.

  • Manually set the correct date and time on the Management Server, or connect the Management Server to a Network Time Protocol (NTP) server.

    Note: If you change the date or time on the Management Server after the Enterprise Manager software is installed, you must reboot the Management Server.

Supported Characters and Length Restrictions in Enterprise Manager and Stealth(aware)

When you are creating components in Enterprise Manager and Stealth(aware)—including configuration, COI, role, and endpoint software file names, as well as user IDs—you must use supported characters and meet the length restrictions.

Note: The first character in a name must be a letter, and only supported characters can be used. The only exceptions are as follows:

  • User IDs can begin with a letter, number, or underscore (_).

  • User Names and Group Names can begin with any character and can include any character (that is supported by Microsoft).

The following characters are supported:

  • Upper and lowercase letters

  • Numbers (0-9)

  • Space ( )

    Note: A space cannot be the first character or last character in a name.

  • Underscore (_)

  • Hyphen (-)

  • Comma (,)

  • Period (.)

    Note: Multiple periods in a row are not supported.

  • Exclamation point (!)

  • Dollar sign ($)

  • Percent sign (%)

  • Ampersand (&)

  • Single quotation mark (‘)

  • Grave accent (`)

  • Parentheses ( )

  • Equal sign (=)

  • At sign (@)

  • Square brackets ([ ])

  • Caret (^)

  • Curly brackets ({ })

  • Tilde (~)

Note: Colon (:) is supported in IPv6 addresses only. You cannot use a colon in a component name.

In addition, COI names and filter set names must be 28 characters or fewer.

Note: COI and filter names are generated automatically in Stealth(aware) to meet these requirements.

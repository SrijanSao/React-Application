Installing the Stealth Endpoint Package on the Management Server

To finalize the initial configuration of your Management Server, you must install the Stealth endpoint package that was automatically created for you by the StealthConfiguration Utility and then configure Windows Server Mode.

 Caution

This action will prevent the Management Server from communicating with any other systems in the environment (besides those for which it includes a clear text filter), because no other systems should be running the Stealth endpoint software at this time.

Do the following:

  1. Browse to the C:\Stealth Files\Software Installs\Packages directory on the Management Server.

  2. Double-click <Admin Configuration Name>AwareWinPkg64-SAO.exe.

  3. If you are prompted to allow the program to make changes to the computer, click Yes.

    The Stealth installer window appears, and you see several messages that the Stealth services are started. When the installation is complete, the installer window closes, and the Stealth Dashboard appears in the task bar.

  4. Do the following to configure Windows Server Mode on the Management Server:

    1. From the Start menu, enter services.msc in the Search box to access the Services.

    2. Locate the Unisys Stealth Logon service in the list of installed services.

      Note: Be sure to locate the Unisys Stealth Logon service (rather than the Unisys Stealth PreLogon service or the Unisys Stealth Protocol service).

    3. Right-click the Unisys Stealth Logon service and select Properties.

    4. On the Properties dialog box, select the Log On tab, and then select the This account option.

      Note: The user account you set to run in Server Mode must be in the administrators group on the local computer. If this is a domain\user name, the domain user must be added to the local computer administrators group.

    5. Enter the user name and password for the Stealth network. (These are the user credentials that determine the Stealth role and COIs when Stealth is enabled on the server.)

      When the user name is set in the account properties and a domain user account is being associated with the service, the administrator must use NT4 format (DOMAIN\username for a domain user or .\username for a local user). The default UPN format (username@www.domain.com) cannot be used because of the problem described in the Microsoft Knowledge Base article 955641. For more information, go to http://support.microsoft.com/kb/955641.

      Note: By default, when using the Browse/Check Names functionality in the Services MMC Snap-in, domain accounts are returned in UPN format. This means that you must change the format of the account name to NT4 style explicitly. This also means that you must know the NetBIOS name of the domain (as the UPN format uses the fully qualified domain name). If you do not know the NetBIOS name of the domain, you should contact your network administrator.

    6. Click OK to close the dialog box.

    7. Click OK when you receive a warning that the changes do not take effect until you restart the service.

    8. Close the Services window.

    9. Reboot the server.

      Note: If you see a dialog box that requests Stealth credentials, you can click Cancel and continue with the reboot.

Stealth is enabled on the Management Server, and the Management Server can now communicate only with systems for which it has a clear text filter configured.

A

Authorization Service

The Authorization Service runs on the Management Server.

The Authorization Service authorizes accounts based on the LDAP groups or individual users added to Enterprise Manager, and it enables users and accounts to access the appropriate COIs, based on the Enterprise Manager configuration.

Administrator

Enterprise Manager Administrators manage the overall configuration of the environment. There are five types of administrators. Each administrator type has different access rights to different pages in Enterprise Manager.

  • Audit Administrator

    • Purpose – Manages logs and monitors the systems added to Enterprise Manager (Management Server, standalone Authorization Servers, Stealth Appliances, SRA Gateways, and Secure Virtual Gateways)

    • Access – Monitoring page and Logs page only

  • Cryptographic Administrator

    • Purpose – Manages Stealth roles, COIs, and filters

    • Access – All pages except the Administration page; however, users in this role are restricted from the following:

      • Users have read-only access to the Monitoring page and all subtabs

      • Users cannot view the License subtab on the Manage page, and cannot install or delete license files

  • Portal Administrator

    • Purpose – Performs all administration functions in Enterprise Manager

    • Access – All pages and functions

  • Portal Operator

    • Purpose – Monitors systems

    • Access – Monitoring page only

  • Service Administration

    • Purpose – Provides for automatic updates performed by services in the environment

    • Access – All pages and functions

  • Stealth Administrator

    • Purpose – Manages the settings for systems in the Stealth network

    • Access – Full access to the Monitoring and Logs pages and the System subtab on the Configure page

      Read-only access to the Provision page and all other subtabs (except System) on the Configure page

C

Channel

Channels define the set of individual device traffic flows either between Profiles (External Channels) or among members of the same Profile (Internal Channels).

Each traffic flow is defined by the source and destination of the IPv4 address and port, by the direction of the traffic, as well as by the TCP or UDP communications protocol.

Communications Policies can be set on traffic flows while building the Stealth(aware) Model, as follows:

  • By defaults defined at the Project or Profile level

  • By the user manually on individual Profiles or traffic flows

Community of Interest (COI)

A group of Stealth endpoints that can communicate securely. You create COIs using Enterprise Manager, and then assign them for use by users and groups.

Configuration

Configurations represent Stealth communication based on the roles, COIs, filters, users, and groups that you define. You can set up one or more configurations in your environment.

E

Enclave

The group of computers that communicate over a Stealth-enabled network.

Endpoint

A computer (workstation or server) that participates in a Stealth-enabled network. This can be a virtual machine or physical server. An endpoint runs the Stealth endpoint software.

N

Network Address Translation (NAT)

An Internet standard that enables a local-area network (LAN) to use one group of IP addresses for internal traffic and a different group of IP addresses for external traffic.

Node

Any device or user identity in the computing environment.

Top

P

Policy

The rule for communications over individual traffic flows. There are three available policies: to allow clear text traffic, to allow Stealth traffic, or to block all traffic.

Profile

A container for devices that perform a similar function; for example, Web Servers, Application Servers, and Database Servers. Profiles are defined on the Stealth(aware) Map through an automated Affinitization of the network traffic flows recorded during Stealth Discovery and then sorted into Profiles using the auto modeling process. You can also manually create a Profile.

Affinitization identifies those devices that include similar traffic flows to other devices, which enables Stealth(aware) to create Profiles that combine similar devices into a single object. During the process of creating Profiles, individual devices (for example, Web Servers) are also auto-classified using a Heuristics Analysis of the traffic data to and from each device.

R

Root

A Profile that is the basis, or root, of a Stealth(aware) Solution.

Top

S

SCIP/IPsec

A combination of the Unisys proprietary Secure Community of Interest Protocol combined with a standard Internet Protocol Security, which is used to encrypt and authenticate each IP packet of a communication session.

Solution

An application stack; for example, an order processing, ERP, or HR application.

The Stealth(aware) Affinitization and auto modeling process identifies those devices and user identities (Nodes) in the network that have frequent communications with each other, and are therefore likely to represent a single application stack. A Solution is comprised of Profiles and their connecting Channels.

Stealth-enabled Network

A network over which data is encrypted using the Unisys proprietary Secure Community of Interest Protocol combined with a standard Internet Protocol Security to encrypt and authenticate each IP packet of a communication session. (This combination of protocols is known as SCIP/IPsec.)

System

A computer that is monitored by Enterprise Manager. This could be a Management Server, standalone Authorization Server, Secure Remote Access (SRA) Gateway, Secure Virtual Gateway, or Stealth 2.8 Appliance.

Top

T

Tunnel

A Stealth endpoint communication path in the environment. There are two types of tunnels: unicast and multicast. A unicast tunnel is a point-to-point communication path between a unique pair of nodes. A multicast tunnel is a multipoint communication path among a set of member endpoints.

Tuples

For Stealth endpoints, a set of COIs, their associated filter sets, and any clear text filters for which the user is authorized.

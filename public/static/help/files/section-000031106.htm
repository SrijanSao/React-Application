Understanding Where to Install the Stealth Discovery Agent

The Stealth Discovery Agent collects additional network traffic data (in pcap format) about your environment, including information about network flows, Node types, and their properties. The Discovery Agent discovers all 5-tuple traffic that is visible from a Node’s network interfaces. (All 5-tuple traffic is captured for each network interface that is enabled and active.) When you install the Discovery Agent on a Windows or Linux Node, the Discovery Agent collects both source and destination information, which means that information about two Nodes is included in each communication flow.

You can install the Discovery Agent on as many or as few Nodes as required to provide a clear model of your environment. For example, you could limit installation of the Discovery Agent to servers in your environment. When workstations communicate with the servers running the Discovery Agent, information about those workstations is then included in the network traffic data. When the network traffic data is imported to Stealth(aware), both the server and workstation Nodes appear in the Stealth(aware) Model.

Best practice: Install the Discovery Agent on one Node in each broadcast domain in your environment. (A broadcast domain is a logical division of a computer network, in which all Nodes can reach each other by broadcast at the data link layer.) On the Node running the Discovery Agent, the NIC must be in promiscuous mode.

Install the Discovery Agent on the appropriate Nodes in your environment to meet the needs of your planned Stealth deployment.

Notes:

  • Only the IP header information is captured by the Stealth Discovery Agents (not the traffic contents).

  • The Stealth Discovery Agent can be installed on virtual machines, and it can discover virtual machines that have unique IP addresses on the network.

Updating and Redeploying the Stealth(aware) Model

 Caution

Before updating the Stealth(aware) Model, ensure Live Feed is Off (from the Import Traffic Data tool).

If you need to update the Stealth(model) after deployment, you can do so at anytime and as many times as required for your environment using Stealth(aware).

For example, you can continue to collect discovery data and import network traffic data into Stealth(aware) from your Stealth-enabled endpoints and from other Nodes in the environment. The Discovery Agent and the Stealth endpoint software can run at the same time. However, if you are gathering information from Stealth-enabled endpoints, you must do one of the following:

  • Stealth-enable the file share that is collecting the network traffic data from the Discovery Agent

  • Create a clear text filter to the file share

Note: Stealth-enabled traffic appears as encrypted traffic in Stealth(aware).

After you import new network traffic data, you can use the modeling tools to make changes to the model; for example:

  • Import new network traffic data

  • Create new Solutions

  • Change the Affinity Level

  • Move Nodes into and out of Profiles

  • Move Profiles into and out of Solutions

When your updated Solutions are ready for deployment, you can use the Deploy tool to redeploy your Solutions.

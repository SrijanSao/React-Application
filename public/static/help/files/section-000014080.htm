Linux Endpoint Requirements

This topic describes the Linux endpoint requirements.

Supported Operating Systems

The following are supported Linux operating systems:

  • Red Hat Enterprise Linux (RHEL)

    • RHEL 6.x, 32-bit and 64-bit

    • RHEL 7.x, 64-bit

    Notes:

    • For RHEL 6.x operating systems, Kernel level 2.6.32-431 or later is required. For RHEL 7.x operating systems, Kernel level 3.10.0-229 or later is required.

    • For RHEL 7.x operating systems, note that the Linux endpoint does not integrate with systemd. The /etc/init.d/stealthd service script is installed, and you should use the standard service commands to start, stop, or restart the stealthd service.

  • SUSE Linux Enterprise Server (SLES)

    • SLES 11.x, 32-bit and 64-bit

      Note: For SLES 11, you must run Service Pack 4 or higher.

    • SLES 12.x, 64-bit

    Notes:

    • For Stealth to operate, the SLES firewall must be disabled. To temporarily disable the firewall (for example, during testing), enter the following command:

      sudo rcSuSEfirewall2 stop

      To disable the firewall permanently (for a production environment), enter the following commands as root, and then reboot your system:

      chkconfig SuSEfirewall2_setup off
chkconfig SuSEfirewall2_init off

    • For SUSE Linux Enterprise Server 11.x endpoints, you must compile and install a required Kernel crypto module. Directions on how to do so are included in Linux Endpoint Initial Installation and Authorization Configuration.

    • For SUSE Linux Enterprise Server 12.x endpoints, Kernel level 3.12.39–47 or later is required.

  • Ubuntu

    • Ubuntu 12.04 LTS, 32-bit and 64-bit

    • Ubuntu 14.04 LTS, 64-bit

    Note: For 64-bit Ubuntu operating systems to operate with Stealth, the AES-NI Kernel module must be disabled. During the endpoint software installation on 64-bit Ubuntu operating systems, the Stealth installer automatically creates the /etc/modprobe.d/blacklist-stealth.conf file, which includes the line “blacklist aesni_intel” to disable AES-NI Kernel module.

Memory Requirements

You must meet the memory requirements for the specific Linux operating system.

For example, to verify that these software packages are installed or to install the required packages, you can enter the following commands, based on your endpoint operating system:

  • Red Hat Enterprise Linux

    • To verify that the software package is installed, enter 

       yum list installed <package name>

    • To install a missing software package, enter 

      yum install <package name>

  • SUSE Linux Enterprise Server

    • To verify that the software package is installed, enter 

       zypper search -i <package name>

    • To install a missing software package, enter 

      zypper install <package name>

  • Ubuntu

    • To verify that the software package is installed, enter 

       dpkg -l | grep <package name>

    • To install a missing software package, enter 

      apt-get install <package name>

Removing Existing strongSwan, openswan, and libreswan Packages

Finally, before installation, you must ensure that you remove the following from your Linux systems:

  • Any preexisting strongSwan, openswan, or libreswan software packages

  • Any related strongSwan, openswan, or libreswan configuration files

Stealth uses a custom version of the strongSwan software, and existing strongSwan, openswan, or libreswan software installations can cause conflicts with the version used by Stealth. To remove strongSwan, openswan, or libreswan from Linux endpoints, enter one of the following commands, depending on the endpoint operating system:

  • For Red Hat Enterprise Linux, enter one of the following commands (depending on whether the system is running version 5.x or 6.x with openswan or running version 7.x with libreswan)

    • sudo yum remove openswan
    • sudo yum remove libreswan

  • For SUSE Linux Enterprise Server, enter one of the following commands (depending on whether the system is running strongSwan or openswan):

    • sudo zypper remove strongswan
    • sudo zypper remove openswan

  • For Ubuntu, enter the following command:

    sudo apt-get remove strongswan
